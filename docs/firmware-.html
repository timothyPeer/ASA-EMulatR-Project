Firmware:

 

Required Files for the Alpha Emulator (ASA)

Here’s how the files you found break down and what the ES40-class emulator actually needs.

 

File

What it contains

When it’s useful in an emulator

Keep / discard

cl67srmrom.exe

Compressed SRM console ROM for ES40 systems with EV67/EV68 CPUs

Mandatory for booting Tru64 UNIX, OpenVMS, NetBSD, Linux, etc. under ES40/AXPbox/es40. Point rom.srm in your config to this file and the emulator will decompress it to rom/decompressed.rom on first run.


slarcrom.exe

Compressed AlphaBIOS/ARC ROM (used only to boot Windows NT)

Optional – supply it if you want to experiment with NT; set rom.arc = "slarcrom.exe" in the config.

clsrmrom.exe

SRM ROM for EV6-based ES40 variants

Not needed for the standard ES40 (EV67/EV68) emulation; use only if you deliberately emulate the earlier EV6 CPU.


clu.exe / mkboot.exe

Loadable-Firmware-Utility (LFU) and diskette-creation helper used on physical ES40s to update firmware

Safe to ignore; they just build boot floppies and add nothing to an emulator.

EV40_V6_1.sys / EV40_V6_2.sys

Firmware for very early EV4/EV40-class workstations

Not compatible with ES40; the emulator can’t use them.

 

 

Within the VS2022 build environment

VS2022 \ Emulator \ AEJ \ Properties \ C++ \ Preprocessor \ CPU_EV=5      // 4=EV4, 5=EV5/21164, 6=EV6/21264, 67, 68, 7…

 

 

EV_SELECT() is purely pre-processor; no run-time cost.

 

In VS 2022 set CPU_EV in Project → Properties → C/C++ → Preprocessor → Preprocessor Definitions.

 

Use a separate configuration or a conditional include if you need to build for several EV levels in the same solution.
