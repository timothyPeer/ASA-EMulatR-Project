AlphaProcessorContext

===============================================================================

AlphaProcessorContext — Characterization

===============================================================================

 

🎯 Overview

-----------

`AlphaProcessorContext` represents the **per-core processor state** of a virtual Alpha AXP CPU.  

It encapsulates everything necessary to describe the current context of a running or halted CPU core, including registers, flags, PC, modes, and special registers.

 

It is owned by a single `AlphaCPU` instance, and is saved/restored on traps, interrupts, or context switches.

 

 

In a Nutshell:

 

•Context = contents of all architecturally visible registers + control state.

•Saved/restored by PALcode & OS during traps & interrupts.

•In emulator: AlphaProcessorContext plays this exact role in software.

 

 

---

 

🧩 Responsibilities

-------------------

✅ Maintains all architecturally visible CPU state.  

✅ Provides access to general-purpose and floating-point registers.  

✅ Stores the Program Counter (PC) and execution mode.  

✅ Holds the processor status (PS) and FP control (FPCR).  

✅ Manages internal processor registers (IPRs).  

✅ Interfaces with stack and exception handling systems.  

 

---

 

📋 Internal State

-----------------

 

| Field                         | Description                                   |

|-------------------------------|-----------------------------------------------|

| `gpr[32]`                     | Integer registers (R0–R31)                    |

| `fpr[32]`                     | Floating-point registers (F0–F31)             |

| `vector[32]` (optional)       | SIMD/vector registers                         |

| `pc`                          | Program Counter                               |

| `fpcr`                        | Floating-Point Control Register               |

| `AlphaProcessorStatus ps`     | Processor Status: mode, IPL, condition codes  |

| `iprBank`                     | Internal Processor Registers (MMU, TLB, etc.) |

| `stackFrames`                 | Shadow stack (if used) for frames             |
