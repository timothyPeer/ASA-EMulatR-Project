Stack - Level 2 (3/4) Support

TulipNIC implements only the L2/MAC and PHY (via MMIO CSRs, DMA, MII).

 MMIO        handled through MMIOManager

 CSRs        TulipNIC

 DMA          

 MII        

 

Above it lives the guest-OS network driver (e.g. “tulip” driver on OpenVMS or Linux), which programs the CSRs, handles interrupts, and publishes a standard network interface (e.g. eth0).

 

The guest’s protocol stack (IP, DECnet, LAT) runs entirely in software above that driver:

 

TCP/IP (IP → TCP/UDP → sockets → apps)

 

DECnet (Network & Transport phases inside the DECnet stack)

 

LAT runs directly on top of Ethernet (it is essentially a datagram service on top of 802.3) and is handled by a LAT daemon in the guest.

 

Because the TulipNIC correctly implements Ethernet 802.3 plus MII for link negotiation, any and all L3/L4 protocols your guest OS supports—including LAT, DECnet, TCP/IP—will run unmodified. The device itself is protocol-agnostic once the frames are handed up through the driver.
