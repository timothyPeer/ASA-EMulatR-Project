Devices

 

The application stack will support this hierarchical device structure.

 

The device hierarchy (Cabinet → Drawer → IODrawer → IOHose → Bus → PCIDevice) is a model of physical/logical containment.

 

 

 

 

[AlphaEmulatorSystem]

|

+--[Cabinet] (cabinetId)

|    |

|    +--[Drawer] (drawerId)

|         |

|         +--[AlphaCPU]*

|         +--[MemoryModule]*

|         +--[IODrawer]

|               |

|               +--[IOHose] (PCI/AGP)

|                      |

|                      +--[Bus] (busId, type)

|                              |

|                              +--[PCIDevice]*   (Bridges, NICs, SCSI, etc)

|                                    |

|                                    +--[Sub-bus] (if Bridge, e.g. PCI-PCI)

|                                    +--[Child Devices] (SCSI, USB, etc)

 |

 +--[Cabinet]

     |

     +--[Drawer]

                    |

                         |

                         +--[PCIDevice] (Slot 2: Adaptec SCSI)

                                |

                                +--[ScsiBusController] (DeviceInterface)

                                        |

                                        +--[ScsiDevice] (e.g., dka0)

 

 

A[DeviceManager / AlphaEmulatorSystem]

   A --> B[Cabinet]

   B --> C[Drawer]

   C --> D[AlphaCPU]

   C --> E[MemoryModule]

   C --> F[IODrawer]

   F --> G[IOHose]

   G --> H[Bus]

   H --> I[PCIDevice]

   I --> J[Bridge? Sub-bus]

   I --> K[DeviceInterface*]

   K --> L[ScsiBusController/Net/USB]

   L --> M[Device]
