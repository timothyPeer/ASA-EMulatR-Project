dt_gfloat — VAX G_Float (64-bit floating-point)

dt_gfloat is a C++ class modeling the VAX G_Float format—a 64-bit floating-point representation used in VAX and supported for compatibility in Alpha AXP. It is similar to IEEE 754 double-precision but has unique encoding details (notably exponent bias and byte ordering).

 

File: dt_gfloat.h

 

Field Layout (bit-level, little-endian memory view)

|  63 | 62        52 | 51              0 |

+-----+-------------+--------------------+

|  S  | Exponent    |   Fraction         |

| (1) |   (11)      |    (52)            |

+-----+-------------+--------------------+

 

 

•S: Sign bit (1 = negative)

•Exponent: 11 bits (bias 1024)

•Fraction: 52 bits (no implicit leading 1 as in IEEE)

 

VAX G_Float is NOT byte-wise identical to IEEE 754 double. It requires careful bit rearrangement during conversion.

Key Methods and Conversions

 

Construction and Assignment

•dt_gfloat() – default (zero)

•dt_gfloat(quint64 raw) – construct from raw 64 bits

•static dt_gfloat fromRaw(quint64) – explicit construction from a bit-pattern

 

IEEE Conversion

•static dt_gfloat fromDouble(double)

•Converts an IEEE 754 double to a G_Float, remapping sign, exponent (with bias adjust), and mantissa bits. Non-finite values (zero, Inf, NaN) are mapped to zero or reserved as G_Float doesn't natively encode all IEEE states.

•double toDouble() const

Reconstructs an IEEE 754 double from the stored VAX G_Float bits (bias and fraction field mapping in reverse).

 

Status/Property Queries

•bool isZero() const

•Checks if the value is exactly zero.

•bool isReservedOperand() const

 

Alpha/VAX compatibility: Returns true for NaN/Reserved operands: All-ones exponent, fraction nonzero (exp == 0x7FF && frac != 0). See ASA Vol I §4.7.7 Table 2-7.

•bool isOverflow() const

•Exponent field is all-ones (0x7FF).

•bool isUnderflow() const

•Exponent field zero, fraction nonzero (subnormal/denormal).

•bool isExactResult(double a, double b, double result) const

Utility for testing whether a result is exactly representable in G_Float.

 

Raw Bits

•quint64 raw() const / setRaw(quint64)

•Direct access/set for internal representation.

 

Display

•QString toHexString() const

•For debugging: printable hex string of the underlying 64 bits.
