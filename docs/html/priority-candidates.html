Priority Candidates

 

 

✅ Tier 1: Top-priority candidates for JIT (high frequency, minimal side effects)

 

Category

Mnemonics (Examples)

Rationale

Integer ALU

ADDQ, ADDL, SUBQ, SUBL, MULQ, UMULH, CMPEQ, CMPULT, CMPLE, CMPLT, AND, ORNOT, BIC, BIS, XOR

Core arithmetic/logical operations, appear in loops, branches

Shift/Mask

SLL, SRL, SRA, ZAP, MSKLL, EXTLL, INSLL

Used in address calculations, bit ops

Compare/Cond Move

CMOVEQ, CMOVNE, CMOVGT, CMOVLT, CMOVGE, CMOVLE

Avoid branches — JIT helps eliminate stalls

Loads/Stores

LDQ, STQ, LDL, STL, LDA, LDAH

Memory ops on hot paths (JIT with cache)

 

✅ Tier 2: Floating-point arithmetic (good JIT gains if pipelined)

 

Type

Mnemonics

Basic Arithmetic

ADDF, ADDG, MULF, MULG, DIVF, DIVG, SUBF, SUBG

Comparison

CMPTEQ, CMPTLT, CMPTLE, FCMOVEQ, FCMOVNE, etc.

Conversion

CVTQF, CVTQL, CVTGF, CVTGQ, CVTQT

Use caution if traps (underflow, inexact) are enabled — then fallback to interpreter.

 

✅ Tier 3: Control flow (static branches, well-predictable jumps)

 

Mnemonics

Notes

BR, BSR, RET, JSR, JMP

Often JIT inlineable if destination is static

BEQ, BNE, BLT, BGE, BLBS, BLBC

Good JIT targets with profiling (branch prediction)

If branch targets are dynamic, insert indirect dispatch or trampoline code.

 

🟡 Lower priority / interpreter-preferred

 

Category

Reason

Memory barriers (MB, WMB, TRAPB)

Rarely hot, better in interpreter

TLB/PAL system (CALL_PAL, REI, TBIS, IMB)

Require context switching or traps

Privileged/special registers (MFPR, MTPR)

Need MMU/supervisor awareness

 

 

Strategy: Precompile these blocks

•Loops with ALU, load/store, branch (hot inner loops)

•Function prologues/epilogues

•DSP-like sequences (shift, mask, multiply, add)

•Memcpy-style loops (LDQ, STQ, ADDQ offset walk)
