Introduction to Symmetric Multi-Processing (SMP) in the ASA-EMulatR project

OVERVIEW

 

Symmetric Multi-Processing (SMP) is the architectural model where multiple processors share a single, coherent memory and I/O space. Each processor runs the same instruction set and has equal access to system resources. The Alpha AXP architecture was designed with SMP in mind, enabling high-performance systems such as the AlphaServer 8400 and ES40 to scale workloads across many CPUs.

For the ASA-EMulatR to faithfully reproduce Alpha hardware behavior and run SMP-capable operating systems (OpenVMS, Tru64 UNIX, Alpha-Linux), support for multiple CPUs must be introduced.

 

 

WHY SMP SUPPORT IS IMPORTANT

 

•  Operating systems such as OpenVMS and Tru64 expect hardware-level SMP features to be present. These include inter-processor interrupts (IPIs), atomic memory operations, and barrier instructions.

•  Realistic SMP allows software testing under conditions of contention, scheduling, and inter-CPU synchronization.

•  SMP emulation enables research into Alpha multiprocessor scaling, cache behavior, and lock contention patterns.

•  For completeness, the emulator must support both single-CPU and SMP configurations as real Alpha hardware did.

 

CURRENT STATE OF THE ASA-EMulatR

 

At present, the ASA-EMulatR models a single AlphaCPU instance. The CPU pipeline, instruction grains, and memory system are architecturally faithful, but execution is restricted to uniprocessor operation. SMP support requires extending this model to multiple concurrent CPUs, each with its own context but operating against a common system image.

We are implementing interface hooks which facilitate inter-processor communication and interrupts necessary to support SMP implementations.  This document-hive will contain specific references to SMP interfaces.
