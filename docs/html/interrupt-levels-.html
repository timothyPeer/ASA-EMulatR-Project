Alpha AXP Interrupt Priority Level (IPL) — Introduction

The Interrupt Priority Level (IPL) field on Alpha AXP systems is a 5-bit architectural value (bits 7:3 in the Processor Status register) that determines whether pending interrupts are delivered to the CPU. Unlike some architectures, Alpha does not assign fixed meanings to individual IPL values—the semantics of each level are defined entirely by the operating system or PALcode (such as OpenVMS, Tru64, or Alpha Linux). The hardware simply compares the interrupt request’s level to the current IPL, and delivers the interrupt only if its level is less than or equal to the current value. This design provides flexible, OS-controlled interrupt masking, enabling efficient SMP and real-time interrupt arbitration.

 

This section documents the canonical bit and mask representations for all 32 IPL levels, as well as code patterns and macros for qualifying and masking interrupts at each level.

Reference tables and examples are provided for both architectural and emulator implementation use.

 

•Alpha IPL is a 5-bit field (values 0–31), bits 7:3 in the PS register.

•On Alpha, IPL is much more abstract:

•The hardware does not assign meaning to specific IPL values.

•All IPL meaning is OS-defined (OpenVMS, Tru64, Linux, Unix.).

•The hardware only uses the IPL to gate interrupt delivery—an interrupt with level ≤ current IPL can be delivered.

•Interrupt requests are “qualified” by the CPU’s current IPL field.

•The PS (Processor Status) register contains the IPL field.

•On Alpha, the hardware treats IPL strictly as a numeric mask.

To mask a single level: mask = INTERRUPT_LEVEL_MASK(level);

To mask multiple levels: mask = INTERRUPT_LEVEL_MASK(2) | INTERRUPT_LEVEL_MASK(5);

To match all levels: mask = ALL_INTERRUPT_LEVELS_MASK; (0xFF for 8 levels)
