Diagram Key Concepts

•IRQController is per CPU (but may also provide system/fabric interfaces).

•AlphaCPU owns an IRQController pointer, sets up all relevant signal/slot connections.

•MMIOManager is global or per bus, fans out device IRQs to the correct controller.

•IPIs use either direct IRQControllerExt or per-CPU controller post methods.

•PAL handling is always done in AlphaCPU context (context save, trap vectoring).

•Memory System is required for TLB shootdown and reservation invalidation via IPI.

•Tick updates: SMPLauncher or system timer should dispatch ticks to all IRQControllers for timer-based interrupts.

 

How MMIO/Device Interrupts Flow

•Device triggers interrupt: Device → MMIOManager::raiseIRQ(cpuId, vector)

•MMIOManager signals IRQController: MMIOManager → IRQController::queueInterrupt(cpuId, vector, ...)

•IRQController sets pending, signals CPU (via Qt signal/slot): IRQController → AlphaCPU::handleInterruptPending()

•AlphaCPU checks, delivers via trap/PAL if allowed: AlphaCPU → (context save, PAL enter, vector)

 

For IPI:

•AlphaCPU::sendIPI(targetCpuId, vector) →

•[IRQControllerExt::postIPI()] →

•IRQController::queueIPI(targetCpuId, vector) →

•AlphaCPU::handleIPI(...)

 

Reference:

•AlphaCPU::attachIRQController()/attachIRQControllers() for setting controller pointers.

•AlphaCPU::setupIRQControllerConnections() for connecting signals/slots.

•IRQController::registerHandler() for per-CPU interrupt handling callbacks.

•MMIOManager (not shown here but in your source) routes device IRQs to IRQController.

•AlphaCPU::handleIPI()/sendIPI()/deliverIPI() for IPI path.

•IRQControllerExt::postIPI() for fabric-aware IPI dispatch.

•AlphaCPU::runExecutionLoop(): periodically checks for pending IRQs/IPIs.

•AlphaCPU::handleInterruptPending()/deliverInterrupt(): does the trap/context save/PAL handoff.
