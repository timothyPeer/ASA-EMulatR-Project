Nan and Reserved Operand Conventions

IEEE quiet NaN is preferred for all conversions that fabricate a NaN (G->IEEE on reserved operand, or other invalid transformations). Preserve sign bit where possible; payload preservation is not required when crossing between VAX and IEEE. For IEEE->G, choose a single canonical reserved operand encoding via dt_gfloat::reservedOperand() to simplify downstream checks.

 

 

Edge cases and recommended tests

 

•FP->INT with src = NaN, +Inf, -Inf -> invalid (and no write-back if traps enabled).

•FP->INT with src just beyond 2^63-1 magnitude -> invalid+overflow flagged.

•INT->FP (double) with src = 2^53+1 -> inexact set; value rounded to nearest even.

•G->IEEE on reserved operand -> quiet NaN, invalid set.

•IEEE->G on NaN/Inf -> reserved operand, invalid set.

•Tiny magnitudes near single subnormal threshold in G->IEEE single path -> underflow/inexact combinations validated.
