SSE4.1 Floating-Point Optimization Reference

This document provides a reference for using SSE4.1 intrinsics to optimize floating-point arithmetic in the Alpha AXP CPU emulator. SSE4.1 (Streaming SIMD Extensions 4.1) was introduced by Intel in 2006 and provides hardware-accelerated floating-point operations with explicit rounding mode control.

 

What SSE4.1 Provides

 

Capabilities

•Fast arithmetic operations** with explicit rounding mode control

•Single instruction** for operation + rounding (no control register modification)

•1-4 cycle latency** vs 30-50 cycles for `fesetround()` approach

•Per-operation rounding** without affecting global FPU state

 

Limitations

•No exception detection** - you must manually inspect results

•No exception trapping** - Alpha trap semantics must be implemented separately

•Single scalar operations** - one value at a time (not vectorized in our use case)

•Requires SSE4.1 CPU support** - must detect at runtime and provide fallback

Rounding Mode Constants

 

_MM_FROUND_TO_NEAREST_INT  // Round to nearest, ties to even (RN)

_MM_FROUND_TO_NEG_INF      // Round toward -∞ (RM)

_MM_FROUND_TO_POS_INF      // Round toward +∞ (RP)

_MM_FROUND_TO_ZERO         // Round toward zero - chopped (RZ)

_MM_FROUND_CUR_DIRECTION   // Use current MXCSR setting

_MM_FROUND_NO_EXC          // Suppress exceptions (should be ORed)

 

**Standard usage:** Always OR with `_MM_FROUND_NO_EXC`:

int rounding = _MM_FROUND_TO_ZERO | _MM_FROUND_NO_EXC;

Complete Usage Example

 

Single-Precision Addition with Chopped Rounding

 

float add_chopped(float a, float b) {

    // Convert float to SSE register

    __m128 va = _mm_set_ss(a);

    __m128 vb = _mm_set_ss(b);

    

    // Perform addition with chopped (toward zero) rounding

    int rounding = _MM_FROUND_TO_ZERO | _MM_FROUND_NO_EXC;

    __m128 vr = _mm_add_round_ss(va, vb, rounding);

    

    // Extract result back to float

    return _mm_cvtss_f32(vr);

}

 

 

Double-Precision Multiplication with Dynamic Rounding

 

double mul_dynamic(double a, double b, FpRoundingMode mode) {

    __m128d va = _mm_set_sd(a);

    __m128d vb = _mm_set_sd(b);

    

    // Map Alpha rounding mode to SSE constant

    int rounding;

    switch (mode) {

        case FpRoundingMode::RN: 

            rounding = _MM_FROUND_TO_NEAREST_INT | _MM_FROUND_NO_EXC; 

            break;

        case FpRoundingMode::RM: 

            rounding = _MM_FROUND_TO_NEG_INF | _MM_FROUND_NO_EXC; 

            break;

        case FpRoundingMode::RP: 

            rounding = _MM_FROUND_TO_POS_INF | _MM_FROUND_NO_EXC; 

            break;

        case FpRoundingMode::RZ: 

            rounding = _MM_FROUND_TO_ZERO | _MM_FROUND_NO_EXC; 

            break;

        default: 

            rounding = _MM_FROUND_TO_NEAREST_INT | _MM_FROUND_NO_EXC;

    }

    

    __m128d vr = _mm_mul_round_sd(va, vb, rounding);

    return _mm_cvtsd_f64(vr);

}

 

CPU Feature Detection

 

CPUID Check

 

#include <cpuid.h>  // GCC/Clang

 

bool hasSSE41() {

    unsigned int eax, ebx, ecx, edx;

    if (__get_cpuid(1, &eax, &ebx, &ecx, &edx)) {

        return (ecx & bit_SSE4_1) != 0;  // ECX bit 19

    }

    return false;

}

 

 

MSVC Version

 

#include <intrin.h>

 

bool hasSSE41() {

    int cpuInfo[4];

    __cpuid(cpuInfo, 1);

    return (cpuInfo[2] & (1 << 19)) != 0;  // ECX bit 19

}

Exception Detection

 

**Critical:** SSE4.1 does NOT detect exceptions. You must manually check results:

 

// After arithmetic operation

dt_sfloat result = dt_sfloat::fromFloat(ieeeResult);

 

// Manual exception checks required:

if (result.isReservedOperand()) {

    // Invalid operation (NaN)

    status->setFlag(StatusBit::InvalidOpOccurred, true);

}

 

if (result.isOverflow()) {

    // Overflow (exponent = 0xFF)

    status->setFlag(StatusBit::OverflowOccurred, true);

}

 

if (result.isUnderflow()) {

    // Underflow (denormal: exp=0, frac!=0)

    status->setFlag(StatusBit::UnderflowOccurred, true);

}

 

// Inexact detection requires comparing exact vs rounded result

double exactSum = static_cast<double>(srcA) + static_cast<double>(srcB);

if (static_cast<double>(result) != exactSum) {

    status->setFlag(StatusBit::InexactOccurred, true);

}

Platform Compatibility

 

•Intel:** Core 2 "Penryn" (2008) and later

•AMD:** Phenom II (2009) and later  

•x86-64:** Nearly universal (2008+)

•ARM/RISC-V:** Not supported, uses `fesetround()` fallback

 

Best Practices

 

1. **Always initialize feature detection** at program startup

2. **Always OR rounding mode with `_MM_FROUND_NO_EXC`**

3. **Manually check all exceptions** after arithmetic

4. **Provide `fesetround()` fallback** for portability

5. **Test on both SSE4.1 and non-SSE4.1 paths**

6. **Don't assume SSE4.1 availability** - detect at runtime

 

Common Pitfalls

 

Incorrect: Assuming exception detection

 

float result = FPArithmetic::add(a, b, FpRoundingMode::RZ);

// SSE4.1 will NOT tell you if overflow occurred!

```

 

Correct: Manual exception checking

 

float result = FPArithmetic::add(a, b, FpRoundingMode::RZ);

dt_sfloat sResult = dt_sfloat::fromFloat(result);

if (sResult.isOverflow()) {

   // Handle overflow

}

 

Incorrect: Forgetting _MM_FROUND_NO_EXC

__m128 vr = _mm_add_round_ss(va, vb, _MM_FROUND_TO_ZERO); // May trigger x86 FP exceptions

 

 

Correct: Suppress x86 exceptions

__m128 vr = _mm_add_round_ss(va, vb, _MM_FROUND_TO_ZERO | _MM_FROUND_NO_EXC);

 

 

Incorrect: Compile-time only check

 

#ifdef __SSE4_1__

   // This only checks if compiler supports SSE4.1, not CPU!

#endif

 

 

Correct: Runtime CPU detection

if (CPUFeatures::hasSSE41()) {

   // CPU actually supports SSE4.1

}
