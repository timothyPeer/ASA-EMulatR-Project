PAL_BASE

PAL_BASE (IPR 3)

Holds the base address for PALcode routines in the kernel super-page region.

•Used to vector to PAL and exception handler code: full address = PAL_BASE + (vector_offset << 13).

•Typically initialized at boot and never changed, but is writable in privileged mode.

readPanicEntry()

writePanicEntry()

PANIC_ENTRY

PANIC_ENTRY (Panic Exception Class Kernel Handler Address)

Synthetic IPR holding the entry point for kernel panic exception handlers.

Considerations.

•Privileged: Readable/writable only in kernel mode (CM=0).

•Only bits [31:0] are significant (32-bit superpage format); full address is composed as:

     0xFFFF_FFFF_8000_0000 | (PANIC_ENTRY & 0xFFFFFFFF)

getRestartVector()

setRestartVector()

RESTART_ADDRESS

RESTART_VECTOR (Restart Execution Address, IPR 0x150 / 336)

This register is written by PALcode or the OS during error handling or special traps. Upon a restart operation, the processor loads the PC from the RESTART_VECTOR register. Holds the address to which execution resumes after a machine check,

 

Considerations.

•correctable error, or certain PALcode routines requiring restart.

oBits 43:13 contain the restart page address (physical page number).

oBits 12:2 contain the offset within the page.

oBits 1:0 are zero (alignment).

TEB

TEB (Thread Environment Block base address)

Synthetic IPR holding the base address of the current thread's environment block.

getThread()

setThread()

THREAD

THREAD (Thread Unique Value, Kernel Thread Address, IPR 27)

Holds the 64-bit address or identifier of the currently executing thread.

 

Considerations.

• Used by OS/PALcode for per-thread data access, kernel thread management, and as a unique context pointer.

•Privileged: Readable/writable only in kernel mode (CM=0).

 

Considerations.

•Future SMP support: define the PS register is QAtomicInteger<quint64> atomic.  

•Platform architectures should implement accessor functions for all changes to the status register.
