Configuration and Setup

Interrupt handling is distributed between several classes:

•AlphaCPU : Implements per-CPU interrupt logic, IPL masking, delivery, exception/interrupt context save/restore, and interface to the IRQController.

•IRQController : Centralizes interrupt queues, arbitration, prioritization, and IPI distribution in SMP systems.

•AlphaProcessorContext  / IprBank : Maintains per-CPU architectural state including PS (Processor Status), IPL, IE (interrupt enable), and SIRR/SISR/SIRR registers for software interrupts.

 

Typical configuration involves:

•Each CPU attached to a shared IRQController for device and IPI signaling.

•IRQController aware of all CPUs for priority arbitration and broadcast.

•Device interrupts (via postDeviceInterrupt), software interrupts (via postSoftwareInterrupt), and IPIs (via postIPI) all funneled to IRQController, then delivered to AlphaCPU(s) as appropriate.
