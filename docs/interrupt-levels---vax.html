VAX Interrupt Priority Level (IPL) — Introduction

The VAX Interrupt Priority Level (IPL) is a 6-bit field (values 0–63) that plays a central role in both hardware and software interrupt arbitration on VAX systems. Unlike Alpha, VAX IPL values have fixed, well-defined semantic meanings assigned by both the hardware and the VMS operating system—with specific levels dedicated to user mode, software interrupts, clock, device interrupts, and machine checks. The IPL field is stored in the Processor Status Longword (PSL), and directly gates which interrupts and code paths are active at any instant.

 

Running code at a given IPL masks out all lower-priority interrupts, ensuring precise interrupt nesting, safe driver routines, and robust AST/software interrupt handling. This page documents VAX IPL value assignments, field encoding, and key differences from Alpha IPL, with practical notes for OS porting and debugging.

 

VAX IPL (Interrupt Priority Level)

•VAX IPL is a 6-bit field (values 0–63).

•Directly accessible and used for both hardware and software interrupt priorities.

•IPL values have fixed semantic meanings in both hardware and VMS software.

•The IPL is stored in the PSL (Processor Status Longword) on VAX.

•The VAX OS and hardware use IPL to gate code execution:

•Code running at IPL=16 or higher masks out all lower-priority device interrupts.

•The OS uses specific IPL “windows” to control AST delivery, softint, device drivers, etc.

 

Specific IPL values are assigned semantic meanings by the hardware and the VMS operating system, such as:

•0: “Normal” (user mode)

•4: Software interrupts (AST)

•8: Clock

•16: Device interrupts

•24: High-priority device interrupts

•31: Console HALT

•39–62: Reserved for special hardware or machine checks

•63: Highest, disables all interrupts
