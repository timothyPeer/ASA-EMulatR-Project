Interrupts

Alpha AXP interrupt classes by type and their assigned IPLs (Interrupt Priority Levels). The official IPL values are mostly in the range 1–31, but not all IPLs are used, and assignment is by DEC conventions (see also PALcode and OpenVMS docs).

 

Interrupt Name

Type

IPL (1-15)

Software

- Summary Register

Summary of all pending software interrupts; read by PALcode or OS for software interrupt dispatch.

- Request Register

Used to request software interrupts from OS components, device drivers, etc.

Asynchonous System Trap (IPL2)

OS mechanism for deferred procedure call, user-level AST delivery in VMS/Tru64.

Passive Release Interrupts (IPL 20-23)

Used by software to unblock or signal specific OS synchronization events.

I/O Device (IPL 20-23)

Hardware: Standard device interrupts; actual assignment depends on platform/hardware. Shared with passive release; arbitration needed.

Interval Clock (IPL 22)

Hardware: System timer interrupt (periodic clock tick), used for scheduling, timeouts, process quantum management.

InterProcessor (IPL 22)

Used in SMP systems for CPU-to-CPU communication (e.g., TLB shootdown, rendezvous, rescheduling).

InterProcessor Register "Typically same as IPI (22) or device (20–23)"

Register-based IPI, implementation dependent.

Performance Monitor (IPL 29)

Triggers when hardware or software performance counters overflow or reach a programmed value. Used for profiling, tuning.

PowerFail Interrupt (IPL 30)

Highest-priority interrupt (except for machine check). Used for orderly shutdown, cache flushing, state saving before power loss.

Machine Check  (IPL 31)

Hardware-detected fatal errors, ECC/memory errors, bus faults. Absolute highest priority. Must be handled immediately.

 

IPL Usage Notes

 

•IPL 0: User mode (no interrupts masked).

•IPL 1–15: Software interrupts, prioritized by software as needed.

•IPL 16–19: (Unused/reserved or platform-specific).

•IPL 20–23: Device and some passive release interrupts (highest hardware interrupt levels).

•IPL 22: Interval clock and interprocessor are often "hard-wired" here (see PALcode/OS conventions).

•IPL 29: Performance monitor (profile) interrupts.

•IPL 30: Power fail.

•IPL 31: Machine check.

 

Interrupt Considerations.

 

•At any time, only interrupts with a higher IPL than the current PS IPL are eligible to be delivered.

•PALcode (in kernel mode) vectors to the correct handler for each IPL/event.

•Most OSes (OpenVMS, Tru64) use IPL 22 as the main clock/device interrupt vector and manage software IPLs via the summary/request registers.

•Passive Release is a software synchronization tool, often used for OS kernel-level signaling (like "wake a waiting thread").

•Interprocessor and Performance Monitor interrupts are central to SMP, profiling, and tuning.
