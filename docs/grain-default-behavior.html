Grain Default Behavior

When no qualifier is specified on an Alpha floating-point instruction, the default behavior is:

 

Default Grain Qualifier Behavior

 

Rounding Mode: /D (Dynamic)

•Uses whatever rounding mode is currently set in FPCR bits [59:58]

•Typically IEEE "round to nearest, ties to even" unless changed by software

 

Exception Handling: Normal FPCR-controlled trapping

•Each exception type (overflow, underflow, inexact, invalid, divide-by-zero) is handled according to its corresponding enable bit in the FPCR

•If FPCR enable bit is SET → trap immediately when exception occurs

•If FPCR enable bit is CLEAR → set sticky status bit but continue execution

 

Implementation:

 

// Default behavior - no qualifier specified

RoundingMode mode = fpcr->getRoundingMode();  // /D behavior

double result = performOperation(srcA, srcB, mode);

 

// Standard IEEE exception checking

if (isOverflow(result)) {

    status->setOverflowOccurred(true);  // Always set sticky bit

    if (fpcr->isTrapEnabled_Overflow()) {

        raiseOverflowException();  // Trap if enabled

        return;

    }

}

 

if (isUnderflow(result)) {

    status->setUnderflowOccurred(true);

    if (fpcr->isTrapEnabled_Underflow()) {

        raiseUnderflowException();

        return;

    }

}

 

if (isInexact(result)) {

    status->setInexactOccurred(true);

    if (fpcr->isTrapEnabled_Inexact()) {

        raiseInexactException();

        return;

    }

}

 

// Continue with result if no traps occurred

writeResult(result);
