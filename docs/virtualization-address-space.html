Virtual Address Space — Overview

This section introduces the virtual address space model as implemented in the ASA-EMulatR, documenting how virtual memory, address translation, and memory protection are achieved to match Alpha AXP (and VAX) architectural expectations. The emulator’s virtual address subsystem is designed to be both architecture-faithful and efficient, supporting robust guest OS operation, accurate exception handling, and JIT-friendly optimizations.

 

Key Subtopics:

 

Memory Management:

•The emulator supports full virtual memory abstraction, providing isolation, protection, and dynamic mapping of guest addresses. Memory management covers RAM, ROM, I/O regions, device windows, and all architectural constraints on valid address ranges.

 

Architectural Registers:

•All key registers involved in address translation are implemented, including the Page Table Base Register (PTBR), Address Space Number (ASN), MMU control/status fields, and privileged internal processor registers (IPRs). These are faithfully emulated to enable correct OS boot, process context switches, and page table traversal.

 

Address Translation and JIT:

•The address translation system integrates tightly with the execution core and JIT engine. The emulator supports efficient, thread-safe TLB (Translation Lookaside Buffer) operations, detects and handles TLB misses and page faults, and supports fast, cache-friendly address translation for JIT-compiled code.

•Translation caches and address validation hooks allow the JIT engine to accelerate guest code execution while preserving architectural correctness.

 

TLBEntry:

•Each translation lookaside buffer (TLB) entry records a mapping from a guest virtual address (and ASN) to a physical page, along with permissions, validity, and usage bits. The TLB subsystem supports per-CPU and global TLBs, supports invalidation and refill per Alpha/VAX rules, and interacts with the page table walker on misses.

 

Page Tables (PTE):

•The emulator implements page table entry (PTE) structures as defined by Alpha and VAX architecture, supporting hierarchical page tables, multiple page sizes, and access/dirty/reference bits. The page table walker performs virtual-to-physical mapping, access validation, and exception signaling as needed, allowing correct OS and process address space management.

 

This section serves as the architectural foundation for all guest memory accesses, exception delivery, context switching, and JIT execution in the emulator. The following subtopics provide detailed documentation and code reference for each aspect of the virtual memory system.
