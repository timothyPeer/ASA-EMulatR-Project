SIRR/SISR (Software Interrupts):

 

Software interrupt request: set SIRR bit, trigger via IRQController.

Delivery clears SISR bit for handled level.

 

 

Interrupt Flow Diagram (Summary)

[Device/Timer/CPU] 

    └──► IRQController::postDeviceInterrupt/postIPI/postSoftwareInterrupt

            └──► Queued in IRQController (per-CPU, by priority)

                    └──► AlphaCPU::checkInterruptDelivery()

                          ├─ If deliverable (IE=1, IPL rules):

                          └──► AlphaCPU::deliverInterrupt()

                                ├─ Save PC, PS, IPL

                                ├─ Set handler address

                                ├─ Raise IPL, mask IE

                                └─ Signal to CPU thread

 

 

 

Code Example:

 

void AlphaCPU::deliverInterrupt(InterruptRequest* request)

{

    wakeIfWaiting();

 

    if (!request || !m_iprBank) return;

 

    // Save current processor state

    quint64 currentPC = getPC();

    quint64 currentPS = m_iprBank->getPS();

 

    // Save INTID (interrupt vector)

    m_iprBank->writeIpr(IPR::INTID, request->vector);

 

    // Save context for REI

    m_iprBank->writeIpr(IPR::EXC_ADDR, currentPC);

    m_iprBank->writeIpr(IPR::EXC_PS, currentPS);

 

    // For software interrupts, clear SISR bit

    if (request->source == InterruptSource::SoftwareSIRR) {

        quint64 sisr = m_iprBank->readIpr(IPR::SISR);

        sisr &= ~(1ULL << request->ipl);

        m_iprBank->writeIpr(IPR::SISR, sisr);

    }

 

    // Update IPL in PS register to mask lower-priority interrupts

    quint64 newPS = currentPS;

    AlphaPS::setIRQL(newPS, request->ipl);

    AlphaPS::setIE(newPS, false);

    m_iprBank->setPSWithSideEffects(newPS);

 

    // Calculate interrupt handler address (SCB-based)

    quint64 handlerAddress = calculateInterruptHandler(request);

    setPC(handlerAddress);

 

    m_inExceptionHandler = true;

 

    emit sigInterruptDelivered(m_cpuId, request->vector, request->ipl);

}

 

PALcode/SCB Vector Handling

 

•On interrupt or exception, PC is set to PALcode or OS handler (SCB) vector:

oE.g., handlerAddress = SCBB + (ipl * 16) (device/software)

oFor IPIs or fatal exceptions, use PAL_BASE + (vector * 64) (see Table 6-2 in ASA Vol II-A).

•Handler may read INTID, EXC_ADDR, etc., then clear the source and REI.

 

Example: Software Interrupt (SIRR/SISR)

 

1.Software sets SIRR bit (requestSoftwareInterrupt)

2.IRQController queues software interrupt.

3.AlphaCPU delivers at next eligible instruction.

4.Handler clears SISR bit for that level.

5.OS returns via REI; interrupts are re-enabled.

 

Clearing/Completing Interrupts

 

•Device IRQ: Device handler must clear source, then inform the controller (e.g., clear device register).

•Software: SISR bit cleared by delivery routine.

•IPI: IPIR bit cleared after service.

•Global: Unmasking interrupts (unmaskInterrupt) allows further delivery.
