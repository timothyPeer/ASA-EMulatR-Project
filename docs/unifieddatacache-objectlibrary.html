UnifiedDataCache

Provides integral support for the L1 I-Cache, L2 D-Cache, L3 Shared (SMP) UnifiedCache, AlphaInstructionCache

 

UnifiedDataCache is best viewed as an instruction word cache (potentially L1 & L2), ensuring fast access to memory without repeated TLB/memory latency. After fetching a word, decode it to an AlphaInstructionGrain via your existing factory or dispatch table. Execute the grain, letting it handle profiling, traps, etc.

 

UnifiedDataCache provides:

 

•Multiple sets & associativity

•Lock-free reads & minimal locking writes

•LRU replacement

•SMP coherency (MESI states, snooping)

•Qt signals for events

•Integration with TLB, memory system, and instruction cache

•Full invalidation, flush, writeback, snoop support

•Per-line metadata (dirty, valid, prefetched, etc.)

•Thread safety with QReadWriteLock

•Lock-free reads via CacheSet

•Configurable associativity, sets, and line sizes

•Integrated with TLB and instruction caches

•Coherency (MESI states, snoop)

•Statistics (hits, misses, utilization)

•Backing store support

•Next/prev level cache hierarchy

 

It supports all the critical functions that the fetch–decode–execute loop and the instruction expect for instruction and data caching.
