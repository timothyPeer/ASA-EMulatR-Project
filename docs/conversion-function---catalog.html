Conversion Function - Catalog

Function catalog (as defined and requested for OptimizedExceptionStatusRegister.h)

 

Notes:

Only the field relevant to the destination should be inspected by the caller (e.g., use intOut for FP->INT, fpOut for INT->FP or FP->FP to IEEE, gOut for IEEE->G).

 

This section documents the helpers available or requested. All are Q_ALWAYS_INLINE, noexcept, and header-only. Where a rounding mode parameter is present, it defaults to round-toward-zero (RZ) when appropriate to match CVTxx variants that specify truncation; otherwise it uses the FPCR mode.

 

1.checkCvttqConversion(double src)

a.Purpose: IEEE double -> signed 64-bit integer (CVTTQ family “T->Q”)

b.Behavior:

i.invalid if src is NaN or infinity or out of signed 64-bit range

ii.inexact if fractional part is nonzero and rounding discards it

iii.overflow if finite magnitude exceeds 64-bit range

iv.underflow is not applicable for FP->INT

v.        Result: sets intOut with rounded/truncated integer; inexact/invalid/overflow as needed.

c.ASA: FP->INT conversion rules, invalid on NaN/Inf, and overflow detection.

 

2.checkCvtdgConversion(double src, FpRoundingMode mode = FpRoundingMode::RZ)

a.Purpose: IEEE double -> VAX G_float

b.Behavior:

i.Maps IEEE NaN or Inf to VAX reserved operand in gOut; sets invalid

ii.Re-bias exponent (IEEE bias 1023 -> VAX G bias 1024) and reorder fraction bits

iii.Detects overflow/underflow per G_float exponent range; sets inexact if rounding changes the result under mode

iv.Result: gOut holds packed dt_gfloat on success; invalid for NaN/Inf; overflow/underflow/inexact per magnitude and rounding.

c. VAX G_float: encoding, reserved operand, exponent bias.

d. ASA: Cross-format conversion expectations when non-IEEE formats are present.

 

3.checkCvtgdConversion(const dt_gfloat& src, FpRoundingMode mode = FpRoundingMode::RZ)

a.Purpose: VAX G_float -> IEEE double

b.Behavior:

i.If src is VAX reserved operand, produce IEEE quiet NaN; sets invalid=true, isNaNInput=true

ii.Re-bias exponent (G bias 1024 -> IEEE bias 1023), reorder fraction bits into IEEE layout

iii.Detect subnormals after conversion; set underflow/inexact if needed

iv.No overflow for finite inputs beyond IEEE range is expected from valid G range; if encountered, mark overflow

v.Result: fpOut is the IEEE double; exception flags reflect mapping and rounding.

c. VAX G_float reserved operand handling -> IEEE NaN.

d.ASA: IEEE NaN quieting and propagation on conversion.

 

4.checkCvtgfConversion(const dt_gfloat& src, FpRoundingMode mode = FpRoundingMode::RZ)

a.Purpose: VAX G_float -> IEEE single (S)

b.Behavior: identical to Cvtgd but narrows to single-precision semantics.

i.Convert G -> double as intermediate, then round to 24-bit precision per mode

ii.underflow for tiny magnitudes landing as IEEE single subnormals/zero

iii.overflow if magnitude exceeds single range

iv.Result: fpOut carries the rounded IEEE result as double (caller narrows to float for storage); exception flags as appropriate.

c.ASA: single-precision rounding and exception signaling rules.

 

5.checkCvtsqConversion(double src, FpRoundingMode mode = FPCR_mode)

a.Purpose: IEEE single/double -> signed 64-bit integer (CVTSQ/CVTTQ family by source width)

i.Notes mirror 1) with the source width considered. If your decoding ensures the correct source width, you may alias to checkCvttqConversion.

 

6.checkCvtqtConversion(qint64 src, FpRoundingMode mode = FPCR_mode)

a.Purpose: Signed 64-bit integer -> IEEE T (double)

b.Behavior:

i.No invalid for any finite integer input

ii.inexact=false for magnitudes <= 2^53 (exactly representable in IEEE double)

iii.inexact=true for |src| > 2^53 due to rounding of LSBs

iv.overflow and underflow do not occur for 64-bit -> double

v.Result: fpOut is the double; inexact set only when necessary.

c.ASA: integer->double exactness threshold at 2^53.

 

7.checkCvtqsConversion(qint64 src, FpRoundingMode mode = FPCR_mode)

a.Purpose: Signed 64-bit integer -> IEEE S (single)

b.Behavior:

i.Similar to cvtqt, but overflow can occur for magnitudes beyond single range

ii.inexact is common except small magnitudes (<= 2^24)

iii.Result: fpOut contains the rounded single value as double; flags per rounding outcome.

c.ASA: integer->single range, rounding, and overflow behavior.

 

8.Optional symmetry: IEEE<->VAX completeness

a.For completeness, provide IEEE single -> VAX G (via double) and integer -> VAX G if your decode paths use those combinations. The same biasing, fraction remap, and reserved-operand handling apply.
