GrainResolver

The `GrainResolver` is the **instruction dispatch registry and lookup engine**  in the ASA-EMulatR.  

 

•Maps raw decoded instruction opcodes & function codes to their corresponding `InstructionGrain` objects, which implement the behavior of each Alpha AXP instruction (including standard, PALcode, and platform-specific grains).

•Provides a clean, efficient way to find the correct handler for a given instruction word during emulation.

 

Considerations.

•Executes via a Singleton (construction and teardown)

•Is SMP Threadsafe (QAtomic)

•Permits one-time initialization and Publication Fence

oBuild registry once, then publish with a release store. Readers assert on an acquire load before using the map.

•Simple Interfaces

oInitialize()

oresolve()

•Freeze after build

•No mutation after threads start

•Destruction Saftey <QGlobalStatic>

 

Boot Sequence:

 

// Single-threaded boot:

auto& gr = GrainResolver::instance();

gr.setSinglePlatform(ProcessorArchMode::Alpha);   // choose EXACTLY one

gr.initialize();                                  // builds and freezes registry

 

// Now start vCPUs; they may call resolve() concurrently, lock-free.

startAllVcpus();

 

 

🧩 Responsibilities

-------------------

✅ Maintain a registry of all available `InstructionGrain` implementations.  

✅ Support platform-aware registration (e.g., Alpha, Tru64, Linux).  

✅ Resolve a 64-bit raw instruction into the appropriate `InstructionGrain`.  

✅ Allow multiple grains to be registered for the same opcode/function

  (if necessary) and pick the best match for the current platform.

 

---

 

📋 Internal Design

-------------------

 

| Component                  | Description                                              |

|----------------------------|----------------------------------------------------------|

| Grain Map                  | Maps `(opcode,function)` → `InstructionGrain*`           |

| Platform Awareness         | Allows grains to be registered per-platform and filtered |

| Registration API           | Registers new grains with their opcode/function/platform |

| Resolution API             | Looks up the grain for given opcode/function at runtime  |

| Singleton (optional)       | Typically implemented as a singleton or owned by CPU     |

 

---

 

📄 Example Responsibilities in Code

-----------------------------------

✅ Registration:

```cpp

gr->registerGrain(new mem_STF_InstructionGrain(), Platform::Alpha);

gr->registerGrain(new pal_MFPR_SISR_InstructionGrain(), Platform::Tru64);

 

Illustrated As:

 

+------------------------+

|    AlphaCPU            |

|------------------------|

| run-loop               |

|    |                   |

|    ▼                   | 

| fetch → decode →       |

| GrainResolver.resolve()|

+------------------------+

          |

          ▼

+------------------------+

|    InstructionGrain    |

|------------------------|

| execute() → semantics  |

+------------------------+

 

 

At runtime:

1️. CPU fetches and decodes an instruction word into opcode + function.

2️. CPU calls GrainResolver::resolve() with opcode + function.

3️. GrainResolver returns a pointer to the registered InstructionGrain.

4️. CPU executes the grain.
