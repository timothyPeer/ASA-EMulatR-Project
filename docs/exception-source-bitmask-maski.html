Exception Source Bitmask Masking

This covers all typical unary, binary, and ternary (FMA, etc.) grains used in Alpha AXP floating-point, following architectural conventions.

 

# Alpha Floating-Point Grain Exception Source Bitmask Map

# (For use in raiseException() and ExceptionInfo)

 

# FORMAT:

#   GRAIN_NAME       ALPHA_EXCEPTION_SRC_BITMASK ARGUMENTS

 

# --- UNARY (single source) ---

# COMPLETE LIST OF ALPHA AXP UNARY FLOATING-POINT INSTRUCTIONS (S, T, Conversion, etc.)

 

CVTQS_*     (m_ra)   # Convert S-float to Quadword Integer

CVTQT_*     (m_ra)   # Convert Quadword Integer to T-float (double)

CVTTQ_*     (m_ra)   # Convert T-float (double) to Quadword Integer (source always F31)

CVTST_*     (m_ra)   # Convert Quadword Integer to S-float (single)

CVTTS_*     (m_ra)   # Convert S-float (single) to T-float (double)

CVTTSU_*    (m_ra)   # Convert S-float to T-float, unsigned variant (less common)

CVTQS_*     (m_ra)   # Convert S-float (single) to Quadword Integer (unsigned variant)

 

SQRTS_*     (m_ra)   # Square Root S-float (single)

SQRTT_*     (m_ra)   # Square Root T-float (double)

 

ABS_S*      (m_ra)   # Absolute Value S-float

ABS_T*      (m_ra)   # Absolute Value T-float

 

NEG_S*      (m_ra)   # Negate S-float

CPYSN_S*    (m_ra)   # Copy Sign Negate S-float (see note below)

CPYSN_T*    (m_ra)   # Copy Sign Negate T-float

 

MOV_S*      (m_ra)   # Move S-float (register move/copy)

MOV_T*      (m_ra)   # Move T-float (register move/copy)

 

# SPECIAL UNARY CONVERSIONS (RARE, but architectural)

CVTQF_*     (m_ra)   # Convert Quadword Integer to F-float (VAX, rare)

CVTGF_*     (m_ra)   # Convert G-float to F-float (VAX, rare)

CVTGFU_*    (m_ra)   # Convert G-float to F-float unsigned (VAX, rare)

 

# PALcode and System/Status ops (PAL, not user FP)

# These are not included in normal FP grain mapping

 

# NOTE: CPYS, CPYSN, CPYSN are "unary" in that only m_ra is significant for exceptions,

# even though they technically also use m_rb for sign bits. In practice, exceptions 

# can only be sourced from m_ra (the magnitude operand), so only m_ra should be 

# included in ALPHA_EXCEPTION_SRC_BITMASK.

 

# If your emulator supports VAX F/G float, or T->S, S->T, etc. conversions,

# add the corresponding conversion grains here.

 

# Summary: For all above, use ALPHA_EXCEPTION_SRC_BITMASK(m_ra)

FCMOVGT_*    (m_ra, m_rb)   # FP Conditional Move Greater Than

FCMOVGE_*    (m_ra, m_rb)   # FP Conditional Move Greater-or-Equal

 

# Note: FCMOVxx instructions use m_ra (source1), m_rb (source2), result goes to m_rc

 

# VAX/DEC compatibility grains (if implemented)

CVTDG_*      (m_ra, m_rb)   # Convert D-float to G-float (if supported)

CVTGD_*      (m_ra, m_rb)   # Convert G-float to D-float (if supported)

# (Rare—include only if your emulator supports VAX FP)

 

# Most other non-computational grains are unary or memory access, not binary.

 

# For all of the above, use:

#   ALPHA_EXCEPTION_SRC_BITMASK(m_ra, m_rb)

 

 

# --- TERNARY (FMA-like, rare) ---

FMA_*                 (m_ra, m_rb, m_rc)  # If/when implemented

 

# --- SPECIAL/RESTRICTED ---

 

# CVTTQ grains: always source F31, so bitmask = (1U << 31)

CVTTQ_*               (31)

 

# FCVTxy grains (VAX, G-float etc.):

FCVTSG, FCVTTG, ...   (m_ra)

# etc.

 

# Memory grains (LDS, LDT, etc.):

# Exception is typically memory access, not operand; only include register if loading causes exception.

 

# --- VECTOR or PACKED (future SIMD support) ---

# For vector ops, use all lanes' indices if any element is exceptional.

 

# --- CONTROL/BRANCH/INTEGER (not FP) ---

# Omit; not handled by ALPHA_EXCEPTION_SRC_BITMASK in FP grains.

 

 

Guidance/Reminders:

 

For unary FP grains:

•Use m_ra (FP or integer source, depending on instruction).

 

For binary FP grains:

•Use m_ra, m_rb.

 

For CVTTQ:

•Always 31 (F31, per Alpha).

 

For memory exceptions:

•If the exception comes from the source register, include it; otherwise, handle as a memory exception (not in FP bitmask).

 

For control/integer grains:

•Not used (handled differently).

 

For vector/SIMD:

•Extend the macro, or use a loop to build the bitmask for all exceptional lanes.
