Internal Processor Register (IPR) Bank

Alpha Emulator — Internal Processor Register (IPR) Bank Documentation

 

🎯 Purpose of the IPR Bank

 

The IPR Bank models the Alpha AXP architecture’s Internal Processor Registers (IPRs) in the emulator. These registers are special-purpose processor state registers that control CPU operation, memory management, interrupts, PALcode, and debugging. They are architecturally distinct from general-purpose integer and floating-point registers.

 

🔷 Role and Responsibilities

 

•Central storage of all IPRs for a given CPU.

•Provides synchronized (QReadWriteLock) read/write access to IPR values.

•Supports architectural defaults and reset to known states.

•Emits Qt signals when registers are updated, enabling UI or diagnostic tools to react.

•Handles special behaviors and side-effects for specific IPRs (e.g., writing to IPIR triggers an interprocessor interrupt).

 

🔷 Usage in VirtualASA

 

Reading & Writing

 

quint64 ps = iprBank->read(IPR::PS);

iprBank->write(IPR::PS, ps | 0x1);

 

or through the Q_INVOKABLE interfaces:

 

quint64 val = iprBank->readIpr(4);  // PS register (ID=4)

iprBank->writeIpr(4, val | 0x1);

 

Example in PAL Instruction

 

// From a PAL grain:

quint64 value = regs->readIntReg(m_ra);

iprBank->write(IPR::IPIR, value);

 

🔷 Structure

 

IprBank

 

•Holds a QVector<quint64> of all defined IPRs (up to 512).

•Maintains a pointer to the owning AlphaCPU for callbacks.

•Protects concurrent access with a QReadWriteLock.

 

Emits signals:

 

sigRegisterChanged(IPR id, quint64 value)

sigBatchChanged()

🔷 Special Considerations

 

Some writes trigger side-effects, implemented in:

 

bool IprBank::handleSpecialWrite(IPR id, quint64 value);

 

Examples:

 

•Writing IPIR signals an IPI.

•Writing TBIA, TBIS, etc., invalidates TLB entries.
