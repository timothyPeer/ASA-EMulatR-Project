ASTEN - Asynchronous System Trap / Enable

ASTEN is a processor register (synthetic, not hardware) that controls whether AST delivery is enabled for each of the 4 processor modes:

 

•Bit 0: Kernel mode (KERNEL)

•Bit 1: Executive mode (EXECUTIVE)

•Bit 2: Supervisor mode (SUPERVISOR)

•Bit 3: User mode (USER)

 

Mask: 0x0F (4 bits, only bits 0–3 valid)

 

Enabling/Disabling AST.

•Setting a bit (to 1) enables AST delivery in that mode.

•Clearing it (to 0) disables AST delivery in that mode.

 

 

Coverage Includes:

 

1.Per-Mode Enable/Disable (bits 0..3)

•You cover this with:

osetASTEnabledForMode(modeBit, enable)

•Dedicated per-mode setters: setASTEnabledKernel(), etc.

oisASTEnabledForMode(modeBit)

oisCurrentModeASTEnabled()

 

2. Bulk Bitmask Operations

•You provide:

osetASTENMask(mask, enable) — sets/clears multiple ASTEN bits

ogetASTENMask() — returns all 4 enable bits (lower 4)

 

3. Get/Set For Current Mode

•Covered by:

osetCurrentModeASTEnabled(bool)

oisCurrentModeASTEnabled()

 

4. Query Full Mask

•getASTEN() const (alias for getASTENMask())

 

5. Atomic/Thread-Safe Operations

•Read/writeIpr() functions are thread-safe or atomic,ready for SMP.

 

See Also.  

 

Alpha Architectural Notes

 

•ASTEN is always 4 bits: one per processor mode (KERNEL, EXEC, SUPV, USER)

•Bit set = AST enabled for that mode

•This matches Digital UNIX, Tru64, and PALcode (see SRM Vol II, Sites & Witek Table 2-7)

•OpenVMS has similar logic, but typically uses a single ASTEN for user mode only (bit 3)

Example:

setASTPendingUser(true);   // Mark AST pending for user mode

setASTPendingKernel(false); // Clear AST pending for kernel mode

 

if (isASTPendingUser()) { /* Deliver AST to user mode... */ }
