Registers Defined — Canonical Register and Bitfield Enumeration

This section documents the full set of architectural and internal registers implemented in the ASA-EMulatR, including all standard Alpha AXP integer, floating-point, and internal processor registers (IPRs). For each register, its enumeration name, offset/number, field structure, default value, privilege level, and key usage notes are provided.

 

Purpose and Approach:

 

•Provides a canonical, architecture-faithful enumeration for all registers in the emulator.

•Serves as the sole source of truth for register access, privilege checks, and bitfield definitions.

•Ensures all emulator and PALcode/OS logic references these enums and bitmasks only (never magic numbers).

 

Register Types Covered:

 

•Integer Registers (R0–R31): General-purpose registers used for all integer and address computation.

•Floating-Point Registers (F0–F31): Full IEEE and VAX G/T/F floating-point support, used by all FP instructions.

 

Internal Processor Registers (IPR): Architecturally defined processor and system registers, including:

 

•PC: Program Counter (current instruction address)

•PS: Processor Status (mode, IPL, interrupt enable, FEN, IV, etc.)

•FPCR: Floating-Point Control Register (exception, trap enable, rounding, FPCC)

•EXC_SUM: Exception Summary Register (fault cause bits)

 

Additional system registers (see IprBank.h and Table 3-4 in the Alpha SRM)

 

Fields, Bits, and Defaults:

 

Each register is defined with:

•Enum name (IPR::PC, IPR::PS, etc.)

•Offset/number (for IPR or system register mapping)

•Bits/fields (bitfield descriptions, field masks and shifts)

•Default value (at power-on/reset)

•Access privilege (AllModes, KernelOnly, PALOnly)

•Usage notes (e.g., “Program Counter”, “Exception Summary”, etc.)

Field and Bit Position Documentation:

•All key fields (mode, IPL, FEN, IV, trap enables, etc.) are enumerated with bit positions and masks, following Alpha SRM and emulator source.

•Bitfield macros and constants (e.g., CM_MASK, IPL_MASK, FEN_MASK, etc.) ensure all code is architecture-faithful and self-documenting.

 

Default Values and Power-on State:

•Each register is initialized to its documented hardware reset value (e.g., PC = 0x20000000, PS = 0x0400...0000).

•See enum, mask, and constant documentation for details.

 

Access Privilege Levels:

 

•Each register is tagged as accessible from all modes, kernel-only, or PALcode-only, enforcing proper architectural privilege checks.

 

Usage and Coding Guidance:

 

•All emulator, PALcode, and OS code should use only these enums and bitmasks for register access, not raw numbers or offsets.

•Macros and enum-based field access allow for future extension, bitfield safety, and maximum maintainability.

 

Reference:

 

Alpha AXP System Reference Manual, Table 3-4, Table 4-7, Ch. 9 (FPCR), Ch. 14 (Exception Summary)

 

IprBank.h (IprBank), FpcrRegister.h (FpcrRegister), SafeMemory.h (SafeMemory), and all associated field/mask headers in the emulator source.
