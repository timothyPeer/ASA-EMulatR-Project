Whether the load originates from user mode, kernel mode, PALcode, or readMMIOPrivileged() helper doesn’t move the boundaries;

 

Privilege code execution decides:

•Can the current page-table entry translate that VA?

•It executes within the same boundaries as the unprivileged code.

 

For EV6/EV7 (43-bit PA) or a board whose firmware moves the PCI hose,  the AlphaSystemManager should register the additional / relocated windows at reset; both the privileged and un-privileged helpers would continue to call MMIOManager::isMMIOAddress() so they automatically pick up the new map.

 

•Faults are handled in the MMU before you ever see the PA.

•Access Faults - access-right bit access failures will raise a protection or access-control fault.
