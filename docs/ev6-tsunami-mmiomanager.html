EV6-Tsunami

EV6/EV67/EV68 (Tsunami) and EV7 (Marvel) maps are larger.

They drive 44 PA bits (16 TB) and reserve PA<43>=1 for all Programmed-I/O.

 

Each PCI/X hose gets its own 1 TB dense + 1 TB sparse pair at addresses 0x8hh0_0000_0000 and 0x8hh4_0000_0000 where hh = hose#.

Core-chip CSR space lives at 0x8060_0000_0000 (EV6) or 0x8600… (EV7).

 

A helper is provide to install windows starting 0x8000 0000 0000 upward when cpuModel >= CPU_EV6.

 

EV6 (21264 + “Tsunami” 21272 core-logic)

 

The CPU itself only distinguishes memory vs programmed-I/O with the top address bit PA [43] (it implements a 44-bit physical address, 0 … 16 TB).

Everything beneath that point is board– or chipset-specific.

 

The Tsunami P-chip breaks the 8 TB PIO half into eight 1-TB hoses with PA [42:40].

Dense windows give a flat, byte-granular view—good for frame-buffer, network, etc.

Sparse windows let legacy 8/16/32-bit devices live on a 64-bit bus; every <size> access is mapped to a naturally-aligned quadword and byte-lane enables.
