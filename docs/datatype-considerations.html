DataType Considerations

The ASA-EMulatR uses specialized floating-point classes to accurately represent all Alpha/VAX floating-point formats and provide safe conversions between them.  dt_gfloat encapsulates the 64-bit VAX G_Float format, providing full support for G_Float arithmetic, storage, and conversion to and from IEEE double-precision. It accounts for the G_Float-specific exponent bias and bit ordering that differ from IEEE.  dt_float is a unified floating-point abstraction capable of representing both IEEE double and VAX G_Float values, with internal tagging to track the underlying format. This class enables seamless conversions, value extraction, and type-safe arithmetic for all supported floating-point encodings.  Safe downcasting is provided through explicit conversion methods—such as .toDouble() and .toVaxG()—with helper routines like .fitsIn64Bits() to ensure precision and range safety. For 128-bit types (dt_tfloat or dt_int128), use these helpers to verify that downcasting to narrower types is valid and will not result in silent data loss.  All conversion routines are explicitly documented in dt_float.h, dt_gfloat.h, and dt_tfloat.h, and conform to the Alpha System Architecture, Volume I (Sections 4.6.1–4.6.2).

 

Qt Data Types: Integration and Rationale

All data types leverage Qt types (quint8, quint16, quint32, quint64, qint8, qint16, qint32, qint64) for clarity, cross-platform correctness, and endian safety. Floating-point conversions use qfloat16, float, double, and long double for widest compatibility.

 

Usage Patterns:

 

•All register and memory operations use fixed-size Qt integer types.

•String conversions for debugging use QString (e.g., dt_gfloat::toHexString()).

•Endian conversions (when mapping to memory or I/O) use QtEndian utilities (qToBigEndian, qFromBigEndian) to maintain architectural fidelity.

•Bitfields, masks, and direct memory operations prefer Qt types for compatibility with QDataStream and QVariant serialization.

Implementation Guidance and Best Practices

•Always validate before downcasting from 128→64 bits.

•Use the provided .fitsIn*Bits() helpers for size checking.

•Use Qt types for all explicit widths, never native C++ types (int, long, etc.) except at API boundaries.

•Prefer .from* and .to* static helpers for conversions.

•For arithmetic, use provided operator overloads in dt_int128 and dt_tfloat where possible.

•For storage, prefer using .low64() or .toDouble() for 64-bit values, or .toHexString() for logs and traces.

Overview

--------

This document defines the canonical conventions, classes, and mechanisms for unified 128-bit data type usage in the Alpha AXP emulator and related tooling. It covers integer and floating-point types, explicit downcasting rules, and Qt data type integration for all sizes and conversion routines. All usage is aligned with the Alpha System Architecture (ASA) and leverages Qt's strong type and endian utilities.

 

Considerations.

•All integer and FP types must use canonical project headers: dt_float.h, dt_gfloat.h, dt_int128.h, dt_tfloat.h
