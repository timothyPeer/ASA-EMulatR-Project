Conversion Function - ResultBundles

Result bundle returned by all conversion checkers

 

Each helper returns a small “conversion result” structure providing the converted value (if applicable) and the exception flags needed by the status register and trap logic.

 

Recommended canonical bundle:

 

struct CvtfpConversionResult

{

    // Destination value in a convenient host type when applicable.

    // For FP->INT, this will be a 64-bit signed integer. For INT->FP or FP->FP,

    // this will be IEEE double unless the helper is VAX-specific and needs dt_gfloat.

    double         fpOut;      // IEEE result when the target is IEEE (S/T), else 0.0

    qint64         intOut;     // Integer result when the target is integer, else 0

    dt_gfloat      gOut;       // VAX G_float when that is the target, else default

 

    // Exception classification computed under FPCR rounding mode (unless instruction overrides).

    bool invalid;              // Invalid operation occurred

    bool overflow;             // Overflow occurred

    bool underflow;            // Underflow occurred

    bool inexact;              // Inexact occurred

 

    // Helper booleans for upstream grains:

    bool isNaNInput;           // Source was NaN (IEEE) or reserved operand (VAX)

    bool isInfInput;           // Source was IEEE +/-inf

    bool isSubnormalInput;     // Source was IEEE subnormal (or VAX denorm if modeled)

};

 

Notes:

 

•Only the field relevant to the destination should be inspected by the caller (e.g., use intOut for FP->INT, fpOut for INT->FP or FP->FP to IEEE, gOut for IEEE->G).

•The helpers do not raise traps. They only classify. Your instruction grain is responsible for setting bits in your Exception/Status register and conditionally raising traps per FPCR enables.

•For S/T destinations, we compute into double and rely on the grain to narrow to single (S) with correct rounding where the ISA requires.
