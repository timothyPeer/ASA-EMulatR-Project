Logical Hardware Model

Logical Model (BusInterface/DeviceManager)

 

•All devices that require MMIO, configuration, system-level query, or interrupt support should implement BusInterface (often as leaf or “active” nodes).

•As part of system init, walk the device tree, and register BusInterface devices with DeviceManager:

•For each PCIDevice, MMIODevice, etc:

odeviceManager->addDevice(static_cast<BusInterface*>(pciDev), family);

 

[DeviceManager]       [SystemBus/MMIOManager]

       |                       |

      (flat registry)    ←→ (dispatch/read/write)

       |                       |

       V                       V

QObject Device Hierarchy (Cabinet-Drawer-IODrawer-IOHose-Bus-PCIDevice*)

       |

      (QObject parent/child, topology, signals/slots)

 

 

Usage:

Any time you want to route MMIO, perform global lookup, or orchestrate device lifecycle: Use DeviceManager + BusInterface.

Any time you want to model, enumerate, probe, or visualize hardware structure: Use the QObject-based device hierarchy.
