About MMU and TLB — Definitions & Roles

 

MMU — Memory Management Unit

-----------------------------------------------------------------------------

The **Memory Management Unit (MMU)** is a hardware component (or software-emulated subsystem) responsible for translating *virtual addresses* generated by the CPU into *physical addresses* in system memory.

 

On Alpha AXP systems, the MMU also:

- Enforces memory protection (read/write/execute permissions).

- Supports multiple address spaces via Address Space Numbers (ASN).

- Generates faults (page fault, protection fault) when translation fails or violates permissions.

- Works closely with the TLB (Translation Lookaside Buffer) to cache translations.

 

In the ASA-EMulatR:

- The MMU is modeled as part of the `AlphaMemorySystem` and `TLBSystem`.

- It consults page tables or predefined mappings to resolve virtual → physical addresses.

 

Typical responsibilities:

    Virtual Address → MMU → (checks TLB → page table) → Physical Address

 

-------------------------------------------------------------------------------

 

TLB — Translation Lookaside Buffer

-------------------------------------------------------------------------------

The **Translation Lookaside Buffer (TLB)** is a high-speed, small hardware cache inside the MMU that stores the most recently used virtual → physical address translations.

 

Why a TLB?

- Accessing full page tables in memory for every memory operation would be slow.

- TLBs dramatically reduce translation latency by caching recent translations.

 

Key characteristics:

- Usually implemented as fully-associative or set-associative cache.

- Has limited number of entries (e.g., 64–256 in typical hardware).

- Entries may include:

    - Virtual Page Number (VPN)

    - Physical Frame Number (PFN)

    - ASN (address space)

    - Permissions (R/W/X)

    - Valid/Dirty bits

 

In the ASA-EMulatR:

- Modeled by `TLBSystem`.

- Each `AlphaCPU` instance typically has its own TLB (per-core).

- On TLB miss:

    - Emulator walks the page table in `AlphaMemorySystem`.

    - Updates TLB with the new translation.

- On TLB hit:

    - Emulator returns the cached physical address immediately.

 

-------------------------------------------------------------------------------

 

MMU & TLB Interaction Summary

-------------------------------------------------------------------------------

- CPU issues a memory access to Virtual Address (VA).

- MMU checks the TLB for VA translation:

    - If hit: retrieves Physical Address (PA) from TLB.

    - If miss: walks the page table, inserts new entry into TLB.

- MMU validates permissions.

- If translation or permissions invalid, raises a fault (page fault, protection fault).

 

-------------------------------------------------------------------------------

 

ASA - Alpha Specific Notes

-------------------------------------------------------------------------------

- Alpha MMU supports:

    - 43-bit virtual address space.

    - Per-process ASN for address space isolation.

    - Large page support (8KB, 64KB, 512KB, 4MB pages).

- TLB entries are tagged with ASN + VPN → PFN mappings.

- TLB is explicitly managed by PALcode and OS during context switches.
