Grain Qualifiers - Saturate

This page documents the trap models and exception checking logic for all floating-point arithmetic instructions in the ASA-EMulatR. The Alpha architecture uses a combination of OptimizedExceptionStatusRegister and FPCR trap enable bits to determine whether a detected exception results in a trap, or is only recorded as a status (“sticky”) bit for later inspection.

 

Example Grains:   ADDS_SUIC_InstructionGrain

           DIVT_M_InstructionGrain

 

Basic Pattern: Each letter in the qualifier combination represents one of the base qualifiers:

 

S = Software completion (deferred trapping)

U = Underflow enable (enhanced detection)

I = Inexact disable (suppress inexact exceptions)

C = Chopped (truncate rounding)

D = Dynamic (use FPCR rounding mode)

M = Minus infinity (floor rounding)

 

/S - Software Completion

•Disables inexact exceptions

•Allows denormal results without trapping

•If overflow/underflow occurs, the OS trap handler fixes it in software

•The instruction completes with a "software-corrected" result

 

/U - Underflow Enable

•Enables underflow trapping

•If a denormal result occurs, trap immediately

•Requires checking the result after computation

 

/SUI - Software + Underflow + Inexact

•Combination of behaviors

•Complex trap enable/disable logic

 

SSE4 Optimizations: FPArithmetic:: 

 

// ==================== FLOAT (Single-Precision) ====================

// Handles /C, /M, /D, /P automatically via roundMode parameter

 

See Also. FP_SSE4_Arithmetic.h

 

 

Rounding Hierarchy: Only one rounding mode can be active:

 

If C is present → truncate

Else if M is present → floor

Else if D is present → use FPCR mode

Else → default behavior

 

Exception Modifiers:

 

S changes normal trapping to deferred/software completion

U forces underflow detection even when FPCR underflow enable is off

I completely suppresses inexact exception reporting

 

So /SUID means: "Use dynamic rounding, enable underflow detection, suppress inexact reporting, and defer all trapping for software completion."

 

 

 

Qualifier

Behavior - Use - Trap Model

Example

/V (Overflow Trap Unconditionally)

Behavior:

Always traps on integer overflow during conversion instructions, regardless of FPCR enables.

 

Use:

Used with instructions like CVTTQ/V (Convert T-floating to Quadword, Overflow).

 

Trap Model:

Unmasked trap: If the overflow occurs, a trap is raised unconditionally, not subject to FPCR trap enables.

auto conv = OptimizedExceptionStatusRegister::checkCvttqConversion(srcA);

if (conv.invalid) status->setFlag(OptimizedExceptionStatusRegister::StatusBit::InvalidOpOccurred, true);

if (conv.overflow) status->setFlag(OptimizedExceptionStatusRegister::StatusBit::OverflowOccurred, true);

 

// Trap on invalid (if enabled)

if (status->hasInvalidOpOccurred() && fpcr->hasFlag(ExceptionCategory::TRAP_INVALID_OP_ENABLE)) {

 ExceptionInfo info;

 quint32 bitmask = ALPHA_EXCEPTION_SRC_BITMASK(31);

 info.setSoftwareRepairBitmask(bitmask);

 info.setPcAtFault(pc);

 info.category = ExceptionCategory::INVALID_OPERATION;

 argCpu->raiseException(info);

 return;

}

// Trap on overflow (if enabled)

if (status->hasOverflowOccurred() && fpcr->hasFlag(ExceptionCategory::TRAP_FLOATING_POINT_OVERFLOW_ENABLE)) {

 ExceptionInfo info;

 quint32 bitmask = ALPHA_EXCEPTION_SRC_BITMASK(31);

 info.setSoftwareRepairBitmask(bitmask);

 info.setPcAtFault(pc);

 info.category = ExceptionCategory::FLOATING_POINT_OVERFLOW;

 argCpu->raiseException(info);

 return;

}

 

/SV (Saturate on Overflow)

Behavior:

Saturates the result to the max or min representable integer value if overflow occurs, and sets the overflow and inexact bits in the exception status register.

 

Does NOT trap!

The instruction does not raise a trap on overflow; it produces a saturated result.

 

Use:

Used with integer conversion instructions, e.g., CVTTQ/SV.

 

Saturation Details:

 

If the input is too large: result is set to 0x7FFF...FFF

If the input is too small: result is set to 0x8000...000

 

Trap Model:

 

Overflow and Inexact are set (sticky), but no trap occurs (even if FPCR overflow enable is set).

if (conv.overflow) {

    conv.value = (srcA > 0)

        ? std::numeric_limits<qint64>::max()

        : std::numeric_limits<qint64>::min();

    status->setFlag(OptimizedExceptionStatusRegister::StatusBit::OverflowOccurred, true);

    status->setFlag(OptimizedExceptionStatusRegister::StatusBit::InexactOccurred, true);

    // Do not trap, just continue

}

 

/C (Chopped/Truncate)

Behavior: Forces rounding mode to "round toward zero" (truncate) for this instruction only, overriding the FPCR dynamic rounding mode.

 

Use: Used with floating-point arithmetic and conversion instructions when you need truncation regardless of current rounding mode.

 

Trap Model: Normal trap behavior applies based on FPCR enables for other exceptions.

// Override rounding mode for this operation

RoundingMode originalMode = fpcr->getRoundingMode();

RoundingMode operationMode = RoundingMode::TOWARD_ZERO;

 

double result = performOperation(srcA, srcB, operationMode);

 

// Check for exceptions with original FPCR enables

if (isInexact(result) && fpcr->isTrapEnabled_Inexact()) {

    raiseInexactException();

}

/D (Dynamic)

 

Behavior: Uses the current rounding mode specified in the FPCR register. This is typically the default behavior when no rounding qualifier is specified.

 

Use: Explicitly indicates that the instruction should use whatever rounding mode is currently set in FPCR bits [59:58].

 

Trap Model: Normal trap behavior applies based on FPCR enables.

// Use current FPCR rounding mode

RoundingMode currentMode = fpcr->getRoundingMode();

double result = performOperation(srcA, srcB, currentMode);

 

// Normal exception handling

if (isOverflow(result) && fpcr->isTrapEnabled_Overflow()) {

    raiseOverflowException();

}

/M (Minus Infinity/Round Down)

 

Behavior: Forces rounding mode to "round toward minus infinity" (floor) for this instruction only.

 

Use: Used when you need floor behavior regardless of current FPCR rounding mode.

 

Trap Model: Normal trap behavior applies based on FPCR enables.

// Force round toward minus infinity

RoundingMode operationMode = RoundingMode::TOWARD_MINUS_INFINITY;

double result = performOperation(srcA, srcB, operationMode);

 

// Normal exception processing

if (isUnderflow(result) && fpcr->isTrapEnabled_Underflow()) {

    raiseUnderflowException();

}

/S (Software Completion/Exception Completion)

 

Behavior: When an exception occurs that would normally trap, the instruction completes with a default result and sets appropriate status bits, but does not immediately trap. The trap is deferred until software explicitly checks or a subsequent floating-point instruction executes.

 

Use: Allows software to handle multiple floating-point exceptions in a batch rather than trapping on each one individually.

 

Trap Model: Deferred trapping - exceptions are recorded but trap delivery is postponed.

bool hadException = false;

 

// Perform operation

double result = performOperation(srcA, srcB);

 

if (isInvalidOp(srcA, srcB)) {

    result = getQuietNaN();

    status->setFlag(OptimizedExceptionStatusRegister::StatusBit::InvalidOpOccurred, true);

    hadException = true;

}

 

if (isOverflow(result)) {

    result = getInfinity(getSign(result));

    status->setFlag(OptimizedExceptionStatusRegister::StatusBit::OverflowOccurred, true);

    status->setFlag(OptimizedExceptionStatusRegister::StatusBit::InexactOccurred, true);

    hadException = true;

}

 

// Store result but don't trap yet

writeResult(result);

 

// Mark for deferred exception processing

if (hadException) {

    status->setPendingSoftwareException(true);

}

/I (Inexact Disable)

 

Behavior: Suppresses inexact exception reporting for this instruction. The inexact condition may occur, but the inexact status bit will not be set and no inexact trap will be generated.

Use: Used when you don't care about precision loss for a particular operation.

Trap Model: Inexact exceptions are completely suppressed - no status bit setting, no trapping.

// Perform operation normally

double result = performOperation(srcA, srcB);

 

// Check all exceptions EXCEPT inexact

if (isOverflow(result) && fpcr->isTrapEnabled_Overflow()) {

    raiseOverflowException();

}

 

if (isUnderflow(result) && fpcr->isTrapEnabled_Underflow()) {

    raiseUnderflowException();

}

 

// Inexact is ignored - do not set status bit or trap

// if (isInexact(result)) { /* SUPPRESSED */ }

/U (Underflow Enable)

 

Behavior: Forces underflow exception checking and reporting for this instruction, even if the FPCR underflow enable bit is not set. Creates a more strict underflow detection than normal.

 

Use: Used when you want to ensure underflow conditions are always detected and reported for critical calculations.

 

Trap Model: Enhanced underflow detection - will trap on underflow regardless of FPCR underflow enable setting.

// Perform operation

double result = performOperation(srcA, srcB);

 

// Standard exception checks

if (isOverflow(result) && fpcr->isTrapEnabled_Overflow()) {

    raiseOverflowException();

}

 

// Enhanced underflow check - forced by /U qualifier

if (isUnderflow(result)) {

    status->setFlag(OptimizedExceptionStatusRegister::StatusBit::UnderflowOccurred, true);

    status->setFlag(OptimizedExceptionStatusRegister::StatusBit::InexactOccurred, true);

    

    // Always trap on underflow with /U, regardless of FPCR setting

    raiseUnderflowException();

}

/SU (Software Completion + Underflow Enable)

Behavior: Enhanced underflow detection with deferred exception handling.

if (isUnderflow(result)) {

    result = getSignedZero(result);  // Software completion default

    status->setUnderflowOccurred(true);

    status->setPendingSoftwareException(true);  // Defer trap

}

/SUC (Software Completion + Underflow Enable + Chopped)

Behavior: Underflow enable + deferred exceptions + truncate rounding.

RoundingMode operationMode = RoundingMode::TOWARD_ZERO;  // /C

double result = performOperation(srcA, srcB, operationMode);

// /SU underflow handling as above

/SUD (Software Completion + Underflow Enable + Dynamic)

Behavior: Underflow enable + deferred exceptions + FPCR rounding mode.

RoundingMode operationMode = fpcr->getRoundingMode();  // /D

double result = performOperation(srcA, srcB, operationMode);

// /SU underflow handling

/SUID (Software Completion + Underflow Enable + Inexact Disable + Dynamic)

Behavior: All of the above but suppresses inexact exceptions.

RoundingMode operationMode = fpcr->getRoundingMode();  // /D

double result = performOperation(srcA, srcB, operationMode);

 

if (isUnderflow(result)) {

    // /SU behavior

    result = getSignedZero(result);

    status->setUnderflowOccurred(true);

    status->setPendingSoftwareException(true);

}

// /I behavior - do NOT check for inexact

/SUIM (Software Completion + Underflow Enable + Inexact Disable + Minus Infinity)

 

Behavior: Floor rounding + underflow enable + no inexact + deferred exceptions.

RoundingMode operationMode = RoundingMode::TOWARD_MINUS_INFINITY;  // /M

double result = performOperation(srcA, srcB, operationMode);

// /SU underflow handling, /I inexact suppression
