Table of Operations (CVT)

Alpha AXP FP Convert Instruction Grains — conversion map (tabs between columns).

Sources: ASA “Alpha AXP System Reference Manual, v6 (1994)”, Chapter 4 “Floating-Point Operate Instructions”: section 4.10 overview/table of contents pages and the specific instruction descriptions cited below. Key pages/lines are shown with each group.

 

Two conversions are performed:

 

•IEEE→Int (CVTTQ)

•Int→IEEE (CVTQS/CVTQT)

•Sfloat↔Tfloat (CVTST, CVTTS)

 

VAX conversions (CVTGQ, CVTQF, CVTQG, and VAX↔VAX context)

 

Notes for columns:

 

•“m_rb read” is the likely helper you’ll call in your AlphaProcessorContext/RegisterBank for the Rb/Fb source. Use your project’s exact names (examples given).

•“Trapping” lists architectural exception classes that can occur for the instruction (not the FP-Disabled fault).

•“Check FPCR flags?” says whether the operation consults FPCR for rounding mode or trap-enable bits and which ones.

•All of these Floating-Point Operate converts require Fa = F31 (per ASA text on each convert)【】【】【】.

Qualifier

Meaning (short)

Effect

Source/Notes

/ID

IEEE, Dynamic rounding

Uses FPCR rounding mode. Traps if enabled in FPCR for: invalid, overflow, underflow, inexact.

Dynamic = from FPCR; this is the “normal” mode.

/IM

IEEE, Minus Infinity

Rounds toward -∞ (minus infinity). Traps if enabled in FPCR for: invalid, overflow, underflow, inexact.

/IC

IEEE, Chopped

Rounds toward 0 (“chopped”). Traps if enabled in FPCR for: invalid, overflow, underflow, inexact.

/IS

Software trap only

No automatic trap; software must check result. Exception bits are set in status register, but no trap.

For implementing custom or deferred error handling.

/N

Integer overflow trap enable

Traps on integer overflow if integer overflow enable bit is set (for e.g., CVTTQ/N, CVTGQ/N).

See “Integer Overflow” handling in ASA.

/II

Inexact trap enable

Traps on inexact if inexact enable bit is set in FPCR (for e.g., CVTTS/II, CVTTQ/II).

Separate from normal inexact detection—enables trap.

 

Supplementary reference for usage:

 

•“The four qualifiers /ID, /IM, /IC, and /IS determine the method of rounding and whether or not exceptions cause traps.” (ASA v6, §4.7.6)

•“/N indicates that the integer overflow enable is honored for this instruction; if not set, overflow merely sets the exception flag.” (ASA v6, Table 4-4)

•“/II selects inexact trapping enable, rather than FPCR default” (ASA v6, Table 4-4)

 

Implementation notes (inline doc pointers for your grains):

 

•Rounding modes for /ID, /IM, /IC are defined in FPCR and qualifier tables (see “Floating-point rounding modes” and “trap modes”)【】.

•Stored-result behavior on overflow/underflow/inexact is centralized in §4.7.6; your grains should route to the same helper used elsewhere for IEEE/VAX store-result semantics【】【】.

•For S↔T: CVTST is exact (no rounding); CVTTS may round and may signal U/II depending on FPCR enables【】【】.

•VAX notes: Conversions involving VAX formats have special invalid-operand rules (reserved operands/“dirty zeros”); ensure your VAX parser flags these as Invalid Operation for CVTGQ and CVTxy where applicable【】【】.
