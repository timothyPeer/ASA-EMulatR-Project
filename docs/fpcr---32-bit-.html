FPCR - 32-Bit

TheFpcrRegister packs all trap, status, rounding, and condition code bits into the lower 32 bits for compatibility and performance.

 

For all architectures, the constructor is always initializes the FCPR raw data as a quint64 64-bit value.

 

Context

Use Which Function

Notes

Alpha hardware/64-bit OS

setRaw64, value64

Use full 64-bit architectural value

VAX/VMS/OSF/Tru64 usermode

setRaw32, value32

Only lower 32 bits meaningful

PALcode (Alpha, OSF/1)

setRaw32, value32

For compatibility, lower 32 only

Hardware register access

setRaw64, value64

Use full 64 bits (if bits defined)

 

Considerations.

•Always use setRaw32() for software/PALcode/VAX-compatible logic, and pass a 32-bit value only.

•Always use setValue32() for software/PALcode/VAX-compatible logic, and pass a 32-bit value only.

•The accessor functions, set32(), and get32() set and return bit-shifted 32-bit values in all cases for 32-bit implementations. The default is 64-bit -- ProcessorArchMode::Alpha
