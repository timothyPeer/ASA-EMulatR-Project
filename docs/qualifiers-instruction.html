Alpha AXP Instruction Qualifier Suffixes

 

Qualifier

Meaning:

Typical Usage:

/M – Masked Exception

All exceptions are masked.Status flags (overflow, underflow, inexact, invalid, reserved operand) are updated, but NO exception/trap is ever raised—even if the corresponding FPCR enable bit is set.Instruction completes; result is always written.

All _M variants, e.g., ADDS_M, DIVS_M, SUBT_M, etc.

/C – Trap on Condition (Conditionally trapped) - Mnemonic: "C" for Checked

Traps (exceptions) are generated if the corresponding FPCR enable bit is set for each exception type (invalid, overflow, underflow, inexact). If the enable bit is not set, only the status bit is updated, no trap is raised. This is the "standard" or "checked" variant in Alpha architecture.

All _C variants, e.g., ADDS_C, DIVS_C, SUBT_C, etc.

/U – Unmasked (Underflow unmasked only)

Underflow traps are always enabled (even if FPCR underflow enable is clear). Other exceptions are masked or follow FPCR enable bits as normal. This is primarily for VAX compatibility and IEEE strict compliance in some cases.

All _U variants, e.g., ADDS_U, DIVS_U, SUBT_U, etc.

/S – Software Trap

All exceptions always cause a trap, regardless of FPCR enable bits. Used mostly in PALcode or software-emulated instructions. Rare in user code.

Not typically exposed in standard Alpha user instructions; mainly PAL. /SU, /SUC, /SUM, /SUD, etc.

 

These are combinations of the above (for extended instructions or for describing behavior in emulation tables):

Qualifier

Usage

/SU

Software trap and Underflow unmasked

/SUC

Software trap, Underflow unmasked, Checked

/SUM

Software trap, Underflow unmasked, Masked

/SUD

Software trap, Underflow unmasked, Double

 

In real Alpha AXP assembly, you almost always see /C, /M, /U (with /C being the default if not specified).
