Cache-Section

      ASA Emulator Configuration Guide — [CACHE/l1], [CACHE/L2], [CACHE/L3]

 

Overview

Each [CACHE/...] section defines the parameters for a hierarchical cache controller, modeling the L1, L2, and L3 cache levels found on Alpha systems. Accurate configuration of cache settings is crucial for realistic emulation of Alpha memory hierarchies, SMP performance, and system coherency.

 

Typical Alpha platforms:

•L1: Split instruction/data (small, fast, per CPU)

•L2: Unified or split, larger, higher associativity

•L3: Optional, very large, used on high-end SMP/GS-series

 

All settings and their units (scale) reflect actual Alpha hardware and ASA Vol. I/II guidance.

 

Key Settings (with Minimums, Types, and Scale)

Type

NumSets

int

Count

Number of cache sets (e.g., 256 for L1, 512 for L2, 1024+ for L3).

>=1; Set per cache size/line; Must be power of two.

Associativity

int

Count

Associativity (number of ways; e.g., 2 for L1, 4 for L2, 8 for L3).

>=1; 2-way typical L1, 4-8-way for L2/L3.

LineSize

int

Bytes

Cache line size in bytes (e.g., 64).

>=16; Power of two; 32–128 typical; 64 is canonical for 21164/EV6.

TotalSize

int

Bytes

Total cache size in bytes (L2/L3 only; e.g., 131072 = 128 KiB, 524288 = 512 KiB).

>=16384 for L2, >=65536 for L3; Not required for L1.

EnablePrefetch

bool

Boolean

Hardware prefetch enabled ("true" or "false").

true (enabled) typical for modern Alpha.

EnableStatistics

bool

Boolean

Enable cache statistics collection ("true" or "false").

true (enabled) for emulator debugging.

EnableCoherency

bool

Boolean

Enable SMP cache coherency ("true" or "false").

true for SMP systems; Set false for uniprocessor.

CoherencyProtocol

string

Text

Coherency protocol name ("MESI", "MOESI").

MESI typical for L1/L2; MOESI on high-end SMP (see ASA Vol. I, §8).

StatusUpdateInterval

int

Count

Interval (cycles or accesses) between status/statistics updates.

>=1; 1000–10000 typical for stats.

ReplacementPolicy

string

Text

Cache line replacement policy ("MRU", "LRU", etc.).

MRU (Most Recently Used) is common for Alpha L1/L2; LRU possible.

EvictionThreshold

int

Count

Eviction trigger for lines (advanced tuning; e.g., 1000).

>=1; Default 1000–10000.

Alpha Architecture Context & Best Practices

•L1 caches are typically split I/D (instruction/data), 8–64 KiB each, 2-way   associative, 32–64B line size, with MESI protocol on SMP.

•L2 caches are unified or split, 64–256 KiB, 4-way or higher, MESI protocol.

•L3 cache (if present) may be 512 KiB to 16 MiB, 8-way or higher, MOESI protocol for large SMP (see GS-series, ES40).

•NumSets, Associativity, and LineSize determine total cache size:     TotalSize = NumSets × Associativity × LineSize (if not directly specified).

•CoherencyProtocol must match hardware and OS SMP support for correct operation.

•ReplacementPolicy affects cache performance, especially on SMP workloads.
