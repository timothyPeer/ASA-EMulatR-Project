Description:

invalidateWhere("Asn", static_cast<quint64>(asn), cpuId);

Invalidate all ASNs for a cpuId or all CPUs.

args: TLBEntry::Asn(), asn (quint64), cpuId (quint16)

invalidateWhere("isDataEntry", true, cpuId);

Invalidate all TLBs which is a DataEntry TBL (true), for cpuId.

args: TLBEntry::isDataEntry(), bool, cpuId (quint16)

invalidateWhere("isInstructionEntry", true, cpuId);

Invalidate all TLBs which is a Instruction TBL (true), for cpuId.

args: TLBEntry::isInstructionEntry(), bool, cpuId (quint16)

 

invalidateWhere("Asn")                        All CPUs

invalidateWhere("asn", 0x2a1, cpuId)        Only cpuId

invalidateWhere("address", virtualPage)        All CPUs

invalidateWhere("address", virtualPage, cpuId)        Only cpuId

 

 

You have a very thorough and flexible set of invalidation functions, addressing almost every common Alpha AXP invalidation scenario.

Here is a summary grouped by function type:

 

Global/CPU-based

•

•invalidateTbiaAllCpus() — Invalidate all TLB entries on all CPUs.

•invalidateTbia(quint16 cpuId) — Invalidate all TLB entries for a specific CPU.

•invalidateAllTLBs_PTE, invalidateAllTLBs_ITLB, invalidateAllTLBs_DTLB — Invalidate all entries by type and CPU.

•You have invalidateByPa_PTE, but not a generic "invalidate any TLB (ITLB/DTLB) by physical page."

 

ASN-based

 

•invalidateByASN(quint64 asn) — All entries with this ASN.

•invalidateByASN(quint16 cpuId, quint64 asn) — All entries for a CPU and ASN.

•invalidateTbiap(quint16 cpuId, quint64 asn) — TBIA for ASN on CPU.

•invalidateByAllAsnByCpuVa(quint16 cpuId, quint64 virtualAddr) -- All ASNs for a CPU and virtual address

 

Single Entry (TBIS/TBISI/TBISD)

•invalidateTbis(quint16 cpuId, quint64 virtualAddr, quint64 asn) — TBIS: Single entry by CPU/VA/ASN.

•invalidateTbisi(quint16 cpuId, quint64 virtualAddr, quint64 asn) — TBISI: Single instruction entry.

•invalidateTbisd(quint16 cpuId, quint64 virtualAddr, quint64 asn) — TBISD: Single data entry.

•invalidateByCpuVaAsn(...) — Exact match, optional PTE.

•BATCH:  invalidateBatch(const QVector<InvalidationRequest>& requests)

o        struct InvalidationRequest {quint16 cpuId;quint64 virtualAddr;quint64 asn; };

 

Type/Property/Condition-based

•invalidateInstructionEntries(quint16 cpuId) — All ITLB entries for a CPU.

•invalidateDataEntries(quint16 cpuId) — All DTLB entries for a CPU.

•invalidatePageTableEntries(quint16 cpuId) — All PTE entries for a CPU.

•invalidateAllPending() — All pending entries.

 

Physical Address

•invalidateByPa_PTE(quint64 physicalPage) — PTEs by PA.

 

Legacy and Range Functions

•invalidateByVa_Legacy, invalidateByVa — By virtual address.

•invalidateByPa_PTE_Legacy — By physical page (legacy).

•invalidateRange_PTE(...) — Range-based PTE invalidation.

 

Compound and Flexible

 

invalidateByTypeAndCpu, invalidateByASNAndType, invalidateByVAAndConditions, invalidateByConditions, invalidateWhere (various overloads), performInvalidation
