IRQL - Interrupt Request Level

Interrupt Request Level (IRQL) — Supported Interface Pattern (Alpha AXP PS Register)

 

 Consistent pattern for reading and setting the IRQL field (bits 7:3):

 

 Code Pattern  

Comments

setIrql( 7 || IRQLFields:: enumeration);

In-place update: set IRQL to 7 in 'ps'

quint8 oldIrql = getIrql();

Extract IRQL from PS register

bool isIrql(7 || IRQLFields:: enumeration)

Returns true if the IRQL is set to this value.

quint64 newPS = withIrql(7 || IRQLFields:: enumeration);

Return new PS with IRQL set to 15

setIrqlAtomic(iprBank.m_ps, 3);

Atomically set IRQL to 3 (QAtomicInteger)

bool atBaseIrql = isIrql(ps, 0);

True if IRQL is 0 (base priority)

bool atSpecificIrql = isIrql(ps, n);

True if IRQL equals value 'n' (0..31)

 

  // Typical workflow:

  quint64 ps = iprBank.getPS();

  quint8 irql = getIrql(ps);       // Read current IRQL

  setIrql(7);                      // Set to 7, in-place

  setIrql(IRQLFields::PASSIVE);    // Set to 0, in-place

  iprBank.setPS(ps);               // Write back updated PS

 

  

 Rationale:

   - Always use AlphaPS::IRQL_MASK and AlphaPS::IRQL_SHIFT for clarity (never magic numbers).

   - Use provided helpers for atomic and non-atomic (in-place) field update.

   - This pattern supports both direct use and programmatic extraction/setting.

   - Matches Alpha architectural intent, ensures safe field updates, and future-proofs for SMP.

 

 Reference:

   - Alpha AXP System Reference Manual, Vol I, §3.2.5.2, Table 3-8

   - Project convention for all PS field set/get helpers (canonical-constexpr.h)
