Memory Barrier Instructions — Overview and Integration

This section documents the Alpha AXP memory barrier instructions and their correct integration in the emulator’s execution core. Memory barrier instructions are critical for enforcing memory operation ordering and system-wide coherency, especially in SMP (multi-processor) environments. They provide architectural guarantees that memory operations before the barrier complete before any operations after the barrier begin, according to the type of barrier invoked (full, load, store, or system-specific).

 

Instruction AlphaInstructionGrains implementing these barriers must call the appropriate memoryBarrier function on the AlphaCPU, specifying both the barrier type and whether the instruction is SMP-aware. This ensures that all guest OS, PALcode, and multiprocessor software running under the emulator observes the expected Alpha memory consistency model.

 

Each instruction variant (MB, WMB, MB2, FETCH, etc.) enforces specific ordering rules:

•Full barriers (MB, RC, RS, TRAPB): enforce both load and store ordering for all CPUs

•Store barriers (WMB): enforce ordering of stores only

•Load barriers (FETCH, FETCH_M): enforce ordering of loads only

•System barriers (WH64, ECB, EXCB, WH64EN): interact with cache, ECC, and prefetch hardware (may require emulator PAL/hardware hooks)

 

Integration into grains is straightforward:

 

•Fetch the PC

•Call AlphaCPU::memoryBarrier(<BarrierType>, smpAware);

•Advance the PC by 4 bytes

 

TRAPB and certain system barriers may require additional emulation, such as pipeline serialization or PALcode hooks.

Integration Guidance for Instruction Grains

AlphaProcessorContext :- context

AlphaCpu :- argCpu

 

Each instruction grain for these operations should call:

 

context->setPC(pc + 4);

argCpu->memoryBarrier(AlphaMemoryBarrierType::<TYPE>, smpAware);

 

Example for MB:

inline void MB_InstructionGrain::execute(AlphaCPU* argCpu, AlphaProcessorContext* context, quint64 rawInstr) const

{

    quint64 pc = context->getPC();

    argCpu->memoryBarrier(AlphaMemoryBarrierType::FullBarrier, true);

    context->setPC(pc + 4);

}

 

Example for FETCH_M:

inline void FETCH_M_InstructionGrain::execute(AlphaCPU* argCpu, AlphaProcessorContext* context, quint64 rawInstr) const

{

    quint64 pc = context->getPC();

    argCpu->memoryBarrier(AlphaMemoryBarrierType::LoadBarrier, false);

    context->setPC(pc + 4);

}

 

Notes

TRAPB additionally enforces instruction serialization. If trap emulation is desired, it should inject a pipeline trappoint.

 

WH64, WH64EN, ECB, EXCB, and RS/RC may also require PAL or hardware hooks — these should call memoryBarrier(SystemBarrier, true) and log prefetch/invalidation as needed.
