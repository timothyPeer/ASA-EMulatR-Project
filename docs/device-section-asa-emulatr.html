Device Section

              ASA Emulator Configuration Guide — [Device.*] Sections

 

Overview

-------------------------------------------------------------------------------

Each [Device.*] section describes a single virtual device on the Alpha system's I/O bus, mapping it to its logical position within the emulated system (PCI, SCSI, IDE, Ethernet, UART, etc.). This mirrors Alpha hardware, where devices are attached via a hierarchy of cabinets, drawers, hoses, buses, and slots (see ASA Vol. I/II and AlphaServer ES40/GS series docs).

 

This section controls SCSI host adapters, disks/tapes, PCI cards, network interfaces, and UARTs. Proper device tree and bus nesting is essential for accurate OS probing, booting, and hardware emulation.

Alpha Bus Nesting and Device Hierarchy

Alpha systems organize I/O with a deeply nested bus structure:

 

cabinet (cabN)

 -> drawer (drwN)

      -> io (ioN)

           -> hose (hoseN)

                -> bus (busN)

                     -> slot (slotN)

                          -> function

 

•Each layer may contain multiple instances (e.g., multiple drawers or hoses).

•Devices are attached at the appropriate slot/function on the bus, reflecting the  physical layout.

•This model allows the emulator to match the device enumeration, boot ordering,  and interrupt routing as seen on actual AlphaServer/GS/DS/AlphaStation platforms.

Example Device Configurations

; PCI SCSI Host Adapter in Drawer 0, Hose 0, Bus 2, Slot 1

[Device.PKA0]

type=SCSI

controller_type=KZPBA

model=QLogic ISP1020

location=cab0/drw0/io0/hose0/bus2/slot1

pci_bus=2

pci_slot=1

pci_function=0

scsi_id=7

irq=56

 

; SCSI Disk on PKA0, ID 0

[Device.DKA0]

parent=PKA0

scsi_id=0

unit=0

image=G:\SomeDir\PaDS20\dka0.vdisk

serial=ST136475LC

irq=auto

Alpha Architecture Context & Best Practices

•Devices must be assigned to unique locations in the bus topology. Refer to your platform’s SRM or service manual for canonical layouts (ES40, DS20, GS1280).

•Child devices (SCSI_DISK, SCSI_TAPE, IDE_DISK) must specify a valid parent controller (e.g., "parent=PKA0").

•All devices must have a unique [Device.*] section key (e.g., PKA0, DKA0, EWA0).

•The "location" field is crucial for proper OS enumeration and matches the DEC/Compaq numbering (cabN/drwN/ioN/hoseN/busN/slotN).

•SCSI IDs (0–7) and unit numbers must match controller type and OS expectations.
