Integration (FDEWI)

This topic describes the integration aspects of the AlphaCPU via the AlphaSMPManager.

 

Fetch / Decode, Execute, Writeback, Invalidate

 

Classes are injected and coordinate the execution of FetchUnit, DecodeStage, ExecuteStage, and WritebackStage in AlphaCPU. Here are the key points:

 

Dependency Injection Pattern

 

Each pipeline stage receives the CPU context, memory system, appropriate caches, and TLB system

•Uses QT-SafePointers (QScopedPointer) for automatic memory management

•Inject dependencies through consistent attach*() methods rather than constructor parameters.

 

TLB Coordination - Translation Buffer Table (TLB)

 

•ITLB: FetchUnit checks instruction TLB for PC translation

•DTLB: ExecuteStage checks data TLB for memory operations

•TLB Misses: Pipeline stages emit signals, CPU handles page table walks

•TLB Invalidation: Coordinated across all pipeline stages and CPUs via signals

 

Cache Hierarchy Integration

 

•L1 I-Cache: Private to FetchUnit for instruction fetching

•L1 D-Cache: Private to ExecuteStage for data operations

•L2 Cache: Shared across CPUs, handles L1 misses

•Cache Coherency: MESI-like protocol with invalidation signals between CPUs

 

Pipeline Coordination

 

•Forward Flow: Fetch → Decode → Execute → Writeback

•Stall Signals: Backward propagation when resources unavailable

•Flush Signals: Branch mispredictions clear pipeline state

•Exception Handling: Centralized through CPU with pipeline notification

 

Memory Operations The ExecuteStage handles:

 

•Load/Store: Normal memory operations with cache hierarchy

•Load-Locked/Store-Conditional: Atomic operations with reservation tracking

•Memory Barriers: Ordering guarantees across the memory subsystem

 

Performance Integration

 

•TLB hit/miss counters

•Cache hit/miss statistics

•Pipeline stall/flush tracking

•Memory operation profiling

 

SMP Coordination

 

•TLB shootdown between CPUs

•Cache coherency protocol

•Inter-processor interrupts

•Shared L2 cache management
