CPU Section

             ASA Emulator Configuration Guide — [CPU] Section

 

Overview

The [CPU] section defines the number of Alpha CPU cores (processors) to be instantiated within the emulator’s virtual SMP environment. This setting models multi-processor Alpha platforms (e.g., ES40, GS1280, AlphaServer 8200/8400) and is fundamental for system scalability, OS SMP boot, and performance.

 

Alpha OSes (OpenVMS, Tru64 UNIX, Digital UNIX) and PALcode support 1–64 CPUs per system, though common hardware configurations were 1, 2, 4, 8, or (on large servers) 16+ CPUs.

Minimum Valid Configuration Example

 

[CPU]

Processor-Count=1

 

Alpha Architecture Context & Best Practices

•The value of Processor-Count should match the intended hardware platform and   OS support. For AlphaServer ES40, 4 CPUs is common; for workstations, 1–2.

•Some Alpha OS bootloaders (including SRM and PALcode) probe the number of  CPUs at boot and expect values to match platform capabilities (see ASA Vol. I,  "Processor and SMP Model").

•Processor-Count > 1 enables SMP mode in OS guests and should be paired with  valid [System] RAM and cache settings for accurate emulation.

•If set lower than 1 or omitted, the emulator will default to single-CPU.
