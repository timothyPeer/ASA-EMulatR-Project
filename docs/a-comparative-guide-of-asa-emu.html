I. Architectural Context

-------------------------

 

Alpha/ASA Bus Hierarchy:

AlphaServer and related DEC hardware (and thus the ASA EmulatR configuration) represent the bus/device tree with explicit, deeply nested topology:

 

   cabinet (cabN) > drawer (drwN) > io (ioN) > hose (hoseN) > bus (busN) > slot (slotN) > function

 

•**Every device** is assigned a *precise physical* or *logical* path.

•This reflects physical system backplanes, modular cabinets, multi-bus/bridge layouts.

•Example (Alpha ES40, GS1280):

 

     cab0/drw0/io0/hose0/bus3/slot1/function0

 

Windows & Linux Bus Models:

Modern Windows and Linux enumerate devices primarily by the **PCI domain:bus:slot:function** tuple, typically expressed as:

 

   domain:bus:device.function

 

•**Domain:** Rarely used outside multi-root/NUMA systems (often 0000).

•**Bus:** PCI bus number (00–FF, hex).

•**Device/Slot:** PCI slot/device number on that bus (0–1F, hex).

•**Function:** Sub-device for multifunction cards (0–7).

•Device mapping is *flat* after PCIe root complexes or bridges.

•Example (common format):  

o    - Windows:  PCI Bus 3, Device 1, Function 0  

o    - Linux:    0000:03:01.0

 

II. Concrete Comparative Examples

 

1. **Alpha/ASA EmulatR Device Configuration**

---------------------------------------------

Example: SCSI controller in ES40 I/O bus layout

 

   [Device.PKA0]

   type = SCSI

   location = cab0/drw0/io0/hose0/bus3/slot1

   pci_bus = 3

   pci_slot = 1

   pci_function = 0

   scsi_id = 7

   irq = auto

 

•**location** specifies the deep system path (matches DEC/Compaq doc diagrams).

•**pci_bus/slot/function** correspond to hardware PCI identifiers.

 

2. **Windows Device Model**

•Windows Device Manager shows device as:

 

     PCI bus 3, device 1, function 0

 

- Registry path (simplified):

 

      HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Enum\PCI\VEN_1077&DEV_1020&SUBSYS... (QLogic SCSI example)

 

•Device detail view:

 

     Bus Number: 3

     Device Number: 1

     Function Number: 0

 

•**No direct "cabinet/drawer/hose"** mapping is visible; only bus/device/function.

 

3. **Linux Device Model**

•Use `lspci` for PCI devices:

 

      $ lspci -nn

      03:01.0 SCSI storage controller: QLogic Corp. ISP1020 Fast-wide SCSI (rev 01)

 

- Tree output (`lspci -tv`):

 

     +-01.0-[03]----01.0  QLogic ISP1020 Fast-wide SCSI

 

- Sysfs path:

 

     /sys/bus/pci/devices/0000:03:01.0/

 

•**Domain (0000)** is usually omitted except on systems with multiple PCI domains (rare on desktops).

•**Again, no explicit nesting**—the topology is flattened after enumeration.

 

III. Mapping: How They Correspond

 

Alpha/ASA          |  Windows/Linux

------------------ |  -------------

cabN/drwN/ioN/...  |  (Not exposed; reflects chassis/backplane)

hoseN              |  (Rare; sometimes maps to a PCI domain or root complex)

busN               |  PCI bus number

slotN              |  PCI device/slot number

function           |  PCI function

 

•The **Alpha hierarchy** is superset; it maps onto the bus/slot/function numbers that Windows and Linux expose, but also encodes *physical* backplane structure.

•The **PCI tuple** (bus, slot, function) is universal and matches the `pci_bus`,  `pci_slot`, and `pci_function` in your ASAEmulatr.ini.

•**Nested fields** (cabinet, drawer, hose) are relevant for:

oAccurate device enumeration/ordering in Alpha OS (OpenVMS, Tru64, SRM).

oLarge SMP/multi-cabinet platforms (GS1280, ES47, etc.)

oHost OS (Windows/Linux) does **not** expose these; physical info is lost.

**ASAEmulatr.ini:**

 

    [Device.PKA0]

    type=SCSI

    controller_type=KZPBA

    location=cab0/drw0/io0/hose0/bus3/slot1

    pci_bus=3

    pci_slot=1

    pci_function=0

    scsi_id=7

    irq=56

 

**Windows/LINUX:**

•Device as seen in Device Manager or lspci:

     PCI bus 3, device 1, function 0

 

VI. Additional Notes

•On real Alpha hardware, *console commands* like `SHOW DEVICE` or `SHOW CONFIG`  will report full cabinet/drawer/hose hierarchy. On Windows/Linux, only bus/slot/function is visible.

•Large Alpha servers use this hierarchy to support >32 PCI buses or >256 slots, far beyond most x86 systems.

•For accurate emulation or OS install/boot testing, configure both the full Alpha location **and** the correct PCI bus/slot/function.
