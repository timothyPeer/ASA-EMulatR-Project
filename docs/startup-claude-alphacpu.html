Startup

Alpha Boot Model (Real Hardware Behavior)

🔹 Single-CPU Boot

•At reset, CPU 0 starts at the PALcode base address (typically 0x21000000).

•Loads SRM/ARC firmware.

•Initializes memory, devices, and starts OS bootstrap.

 

🔹 SMP Boot

•CPU 0 is the bootstrap processor (BSP) — it begins execution from 0x21000000.

•Other CPUs (APs = Application Processors) are initially halted.

•Once the OS (e.g. VMS, Tru64) initializes, it issues START_CPU(cpu_id, pc) commands:

•This sets the starting PC for each AP

•Each AP starts executing at the specified PC address (usually a spin loop or trampoline)
