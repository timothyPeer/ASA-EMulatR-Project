SafeMemory

🎯 Overview

-----------

`SafeMemory` is the core abstraction of **physical RAM** in the ASA Alpha Emulator.  

It provides a simple, thread-safe, bounds-checked interface for reading and writing

raw bytes, words, and quadwords from the emulated system’s main memory.

 

It is called *safe* because:

- it validates memory accesses to prevent out-of-bounds errors,

- it guards against concurrent access issues (if required),

- it detects invalid or uninitialized regions.

 

It backs the `AlphaMemorySystem` and ultimately serves as the storage layer

for all RAM accesses (data & instructions).

 

---

 

🧩 Responsibilities

-------------------

✅ Allocate and manage the emulator’s physical memory space (RAM).  

✅ Provide efficient, bounds-checked access to memory contents.  

✅ Support reads & writes of 8, 16, 32, and 64-bit values.  

✅ Detect and report invalid accesses (out of bounds or unmapped).  

✅ Optionally support memory initialization and region mapping.  

 

---

 

📋 Internal Design

-------------------

 

| Component                 | Description                                             |

|---------------------------|---------------------------------------------------------|

| Memory Buffer             | A contiguous `std::vector<uint8_t>` or equivalent array |

| Size                      | Size of physical memory in bytes                        |

| Bounds Checking           | Ensures reads/writes are within allocated range         |

| Region Mapping (optional) | Defines valid/invalid regions for advanced use          |

| Thread Safety             | May use mutexes/locks if needed                         |

🚀 Why It’s Important

1.Provides a clear separation between address translation (AlphaMemorySystem)and storage (SafeMemory).

2.Detects and prevents invalid memory accesses.

3.Simplifies debugging by isolating out-of-bounds errors.

4.Ensures RAM is modeled faithfully.
