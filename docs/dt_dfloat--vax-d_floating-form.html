dt_dfloat -VAX D_floating format (64-bit, bias 128, 55-bit fraction)

VAX D_floating (double precision, 64 bits) suitable for Alpha and VAX emulation, architecturally faithful to VAX and Alpha bit layouts.

•Follows the style of your dt_ffloat, dt_gfloat types

•Implements lossless bit/field mapping for register/memory ops

•Handles bias, reserved operand, and architectural peculiarities

•Provides fromRaw, raw, fromDouble, toDouble as with your other types

 

// VAX D_floating format (bit layout, big-endian numbering):

//   63     | 62:55   | 54:0

//   sign   | exponent| fraction (55 bits, MSB implied, no explicit hidden bit)

// - Exponent: 8 bits, bias 128 (same as F_floating)

// - Fraction: 55 bits (MSB implied, fraction is left-aligned in register)

// - VAX byte/word order is non-IEEE; use raw bits for emulation.

 

Key Points:

•All conversions preserve bit fields as per ASA Table 2-7, Figure 2-8.

•Use this class for memory/register transfer, CVTDQ/CVTQD, LDQ_D/STQ_D, etc.

•Use the fromGFloat() and toGFloat() for internal Alpha register mapping.

•For direct computation, use toDouble()/fromDouble() with care: not all VAX D_floating patterns are valid IEEE.

 

How D_floating Data is Stored

1. When D_floating data is loaded from memory:

•The 64-bit VAX D_floating value (memory layout) is read from memory.

oIt is immediately converted ("expanded") to Alpha G_floating canonical form before being stored in the floating-point register.

oExponent: D_floating exponent (8 bits, bias 128) is expanded to G_floating exponent (11 bits, bias 1024) using: G_exp = (D_exp == 0) ? 0 : (D_exp + 896)

oFraction: D_floating fraction (55 bits) is right-shifted to fit the 52-bit G_floating fraction field.

oSign: Mapped directly.

•The register file never holds a native VAX D_floating value—only G_floating.

 

2. When a D_floating value is stored to memory:

•The 64-bit G_floating register value is converted ("contracted") to D_floating using the reverse mapping:

oExponent: D_exp = (G_exp == 0) ? 0 : (G_exp - 896)

oFraction: G_floating fraction (52 bits) is left-shifted to form the 55-bit D_floating fraction field.

oSign: Mapped directly.
