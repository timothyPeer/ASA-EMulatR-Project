64-bit Floating Point: dt_gfloat (VAX G_Float) and dt_float (Unified)

 

Example usage and downcasting guidance are shown below.

 

•dt_gfloat encapsulates VAX G_Float (64-bit, similar to IEEE double, different exponent bias and ordering).

•dt_float is a unified floating-point class representing either IEEE double or VAX G_Float, with dynamic tagging.

 

Example:

 

#include "dt_gfloat.h"

dt_gfloat g = dt_gfloat::fromDouble(2.71828);

double ieee = g.toDouble();

 

or

 

#include "dt_float.h"

dt_float f = dt_float::fromIEEE(2.0);

double d = f.toDouble();

dt_gfloat vax = f.toVaxG();

 

Downcasting:

For 128→64 bit conversion, use the .toDouble() or .toVaxG() methods; the conversion will use the low 64 bits and ignore overflow or high-precision loss by design. Downcast only when .fitsIn64Bits() or the value is known to be in range.

 

See: dt_float.h and dt_gfloat.h

Reference: ASA Vol I, 4.6.2 “G_Float” and 4.6.1 “F_Float”

 

Downcasting: 128-bit to 64-bit, 32-bit, 16-bit, 8-bit

•The classes provide explicit conversion routines for downcasting, with range-check helpers:

•dt_tfloat::fitsIn64Bits() returns true if downcasting is safe (high == 0)

•dt_tfloat::toInt64(), .toUInt64(), .toDouble() for numeric conversions

•dt_tfloat::fitsIn32Bits(), .fitsIn16Bits(), .fitsIn8Bits() for safe truncation checks

•All downcasts are truncating: high bits are discarded, user is responsible for semantic checks

 

Example:

 

if (tf.fitsIn32Bits()) {

    qint32 v = tf.toInt32();

}

 

•For dt_int128, always extract via low64(); sign-extension or overflow must be checked by caller.
