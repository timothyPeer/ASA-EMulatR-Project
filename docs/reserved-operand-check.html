ALPHA_FP_RESERVED_OPERAND_CHECK - Reserved Operand Check

Typical Macro Usage for Reserved Operand/NaN Checks

 

In most implementations, this macro is intended to check all operands (A, B, possibly C) for being NaN or reserved operand, and to trap if any are.

 

The prototype is often:

 

ALPHA_FP_RESERVED_OPERAND_CHECK(srcA, srcB, status, regA, regB, pc, argCpu)

 

Where:

 

•srcA, srcB are the actual floating-point values (as doubles or floats)

•regA, regB are their respective register indices (for bitmask construction)

•status is the OptimizedExceptionStatusRegister

•pc, argCpu are for context and trap

 

If the Instruction is Unary (e.g., SQRTT):

 

•srcA is valid (actual operand, e.g., double srcA = ...)

•srcB is not used, pass 0  as a placeholder.

 

 

Macro Considerations:

•The macro should ignore srcB if the instruction is unary. If the macro always checks both operands for NaN/reserved, passing 0 as a double (which is never NaN) means “no error”—that’s safe for unary. If the macro expects only to check srcA in unary context, this is entirely correct and is common in canonical DEC/VAX/Alpha-style emulator codebases.
