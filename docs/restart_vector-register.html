RESTART_VECTOR Register

Holds the address to which execution resumes after certain exceptions, faults, or PALcode-handled events (e.g., hardware interrupts, faults, or explicit PAL operations requiring the CPU to “restart” or “continue” an instruction sequence).

 

Mechanism:

When an exception or interrupt occurs, PALcode saves state (including the current PC and PS) and may set the RESTART_VECTOR to a handler or the next instruction to execute upon return from the handler.

 

•Typical Use Cases

•

•Returning from a hardware interrupt.

•Retrying a faulted memory access after handling a page fault.

•Restarting user code after a signal or system call.

•System "double fault" and "panic" handling (though PANIC_ENTRY Registry is the canonical fatal case).

 

Platform Architecture

Implementation

OSF/1 / Tru64

RESTART_VECTOR is always present in OSF/1 PALcode as a synthetic (PAL-maintained) register.

 

It is accessible via privileged PAL instructions and is not directly modifiable by user mode or unprivileged code.

 

Format is typically a 64-bit physical or virtual address, depending on context.

OpenVMS

RESTART_VECTOR is not used or exposed in the same way. VMS handles instruction restart/resume via trap frames and explicit PC restoration, not a dedicated synthetic register.

Linux/BSD

If using OSF/1 PALcode, RESTART_VECTOR is present as expected.

 

Linux PALcode or other custom PALcodes may not implement it.

 

Implementation Note:

•Settable/Readable only in privileged mode (kernel or PAL).

•Not present in hardware; it's always emulated in PALcode or the emulator's synthetic register file.

•Zero on reset or after fatal errors.

•May be set to a superpage-mapped or physical address, depending on the OS/PALcode design.

VECTOR DEPOSIT-VALUE:

On most critical faults:

 

RESTART_VECTOR is set to zero (0x0000_0000_0000_0000). RESTART_VECTOR_UNRECOVERABLE_FAULT

•This is a signal to the kernel’s fatal handler (“panic handler”) or PALcode that no valid instruction can be restarted.

•On recoverable faults (e.g., correctable memory error, retryable fault):

 

RESTART_VECTOR is set to the instruction address to retry.

•Typically, this is the faulting PC (program counter), or the address of the “continuation” point, possibly advanced for completed instructions.

•On fatal ("panic") class exceptions:

 

RESTART_VECTOR is always cleared (set to zero).

•OS kernel or PAL handler will not attempt to return to user/kernel code, but will instead transfer control to the PANIC_ENTRY or equivalent handler.

 

Example for later inclusion:

if (isUnrecoverableFault) {

    setRestartVector(RESTART_VECTOR_UNRECOVERABLE_FAULT);

} else if (isRetryableFault) {

    setRestartVector(faultingPC); // set at runtime

}
