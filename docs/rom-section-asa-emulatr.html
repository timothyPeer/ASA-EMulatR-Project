ROM Section

              ASA Emulator Configuration Guide — [Rom] Section

 

Overview

The [Rom] section defines all ROM (Read-Only Memory) and firmware image paths required for system initialization and device emulation in the Alpha emulator. These images provide PALcode, console, and hardware-level microcode for the target platform, ensuring architectural fidelity for boot, device probe, PCI enumeration, and firmware-driven diagnostics.

 

Correct selection and configuration of ROM files is essential for booting real OpenVMS, Tru64 UNIX, or SRM/ARC console environments in the emulator.

 

Settings must reflect the required binary/HEX images for the chosen Alpha model, as described in the Alpha System Architecture and the system's SRM or hardware documentation.

 

Key Settings (with Minimums, Types, and Scale)

Type

Description / Example

Minimum / Notes

clipperFileName

string

Filename

Path to the Clipper ROM binary (e.g., "clipper.bin").

Required for many AlphaServer and ES40/GS-series systems. Must be a valid file path and readable at boot time.

HostProcessorModuleFirmwareFile

string

Filename

Path to Host Processor Module firmware (HEX) (e.g., "hpmrom.hex").

Optional but strongly recommended for accurate CPU/board initialization; Use the file matching your board revision.

PCIBusModuleFirmWare

string

Filename

Path to PCI Bus Module firmware (HEX) (e.g., "pbmrom.hex").

Optional; Needed for systems with a modular or field-replaceable PCI host bridge.

SystemModuleFirmwareFile

string

Filename

Path to System Module firmware (HEX) (e.g., "scmrom.hex").

Optional; For board controller/logic; check platform documentation.

IntelHexLoaderFile

string

Filename

Path to a generic Intel HEX loader (e.g., "clipper.hex").

Optional; Used for certain bootstrap, field diagnostics, or console flash.

 

Minimum Valid Configuration Example

 

[Rom]

clipperFileName=clipper.bin

 

Alpha Architecture Context & Best Practices

•All file names must point to valid and accessible binary or HEX images. These   are typically extracted from real Alpha hardware, DEC/Compaq firmware updates,   or provided with Alpha SRM/ARC console kits.- For platforms such as ES40 and GS1280, the Clipper ROM is mandatory and contains board-specific PALcode required to bring the CPU out of reset and

•  initialize the memory and I/O buses (see ASA Vol. II, "Console Firmware").

•HostProcessorModuleFirmwareFile, PCIBusModuleFirmWare, and   SystemModuleFirmwareFile are optional but improve architectural fidelity and are often required for advanced PCI, SMP, and diagnostics support.

•IntelHexLoaderFile is rarely required for normal boot but may be used for  debugging or loading secondary images.
