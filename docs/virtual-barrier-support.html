Virtual Barrier Support

 

These are defined by virtual address ranges, influenced by mode bits, MMU settings, and page table base register contents.

 

Space

Name

Description

P0

User space

Virtual addresses accessible in user mode, grows upward

P1

Stack space

User-mode stack space, grows downward

S

System space

Kernel-mode accessible addresses, shared between processes and system data

 

 

Address Ranges and Access Modes

Each space is associated with a segment of the virtual address space:

 

Space

Virtual Address Range

Accessible From

P0

0x0000_0000_0000_0000 → 0x0000_3FFF_FFFF_FFFF

User & Kernel (depending on permissions)

P1

0x0000_4000_0000_0000 → 0x0000_7FFF_FFFF_FFFF

User (stack)

System

0xFFFF_FFFF_8000_0000 → 0xFFFF_FFFF_FFFF_FFFF

Kernel mode only

 

Interface Dependencies

 

AlphaProcessorContext

Holds current processor mode (user/kernel/supervisor), Provides access to: getCurrentMode()getP0Base(), getP1Base(), getS0Base,(), getPTEBase(), etc. ; Influences the interpretation of ASNs, access control, and virtual-to-physical translation logic

AlphaMemorySystem

Knows how to translate virtual address ranges based on the current mode Uses the page table base register and ASN to determine the appropriate region (P0, P1, S). Required fields; -getP0PageTableBase(),-getP1PageTableBase(), -getSystemPageTableBase(), -determineAddressSpace(quint64 va, ProcessorMode mode)

TLBSystem

Must tag TLB entries with the correct ASN + space, P0/P1 entries are per-process, System entries are global or per-ASN, depending on VMS/Tru64 settings

SafeMemory

Provide functions to validate kernel-only accesses, Support read access from page tables in S-space
