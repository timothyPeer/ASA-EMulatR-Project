System Section

The [System] section in the ASAEmulatr.ini configuration file defines the virtual hardware board and system-level characteristics for the Alpha emulator. These settings determine the base resources available to all emulated CPUs, MMU, memory system, and top-level board model.

 

Settings and their scale should match Alpha System Architecture recommendations (see ASA Vol. I: Memory Management, System Configuration chapters).

 

Key Settings (with Minimums, Types, and Scale)

 

Key Name

Type

Scale

Description / Example

Minimum / Notes

MemorySizeGB

int

GiB

Total system RAM size (e.g., 32 = 32 GiB RAM). Used to size the system's physical memory region.

1 GiB minimum recommended; Valid range: 1–1024; Values <1 GiB may cause OS boot failures.

hw-Model

string

Text

Board hardware model identifier (e.g., "ES40", "AlphaServer 1000A"). Matches typical DEC/Compaq platform names.

Required (no default); Used for platform reporting, logging, and console banner.

hw-Serial-Number

string

Text

Board or chassis serial (e.g., "AX122312341243134").

Recommended; Used for virtual asset tracking and emulated firmware.

Coherency-Cache

int

KiB

L1 I-cache coherency size, specified in kilobytes (e.g., 2048 = 2 MiB).

1024 (1 MiB) minimum; 2048 KiB (2 MiB) typical; Value must be power of two.

Platform-Ev

int

Number

Alpha EV processor family (e.g., 6 = EV6/21164, 5 = EV5/21164, 4 = EV4/21064).

Must match target CPU model and platform features; 4 (EV4), 5 (EV5), 6 (EV6).

PTE-PageSize

int

Bytes

MMU page size in bytes (power of two, e.g., 8192 = 8 KiB pages).

8192 (8 KiB) typical; Minimum: 4096 (4 KiB); See ASA Vol. I, §5.2.3.

CPU_FREQUENCY_HZ

int

Hz

CPU frequency in Hertz (e.g., 500000000 = 500 MHz).

Value should match model hardware spec;

Range: 100000000 – 1500000000;

500000000 (500 MHz) is typical for ES40/21164.

 

Minimum Valid Configuration Example (for AlphaServer ES40, EV6)

[System]

MemorySizeGB=4

hw-Model=ES40

hw-Serial-Number=AX987654321

Coherency-Cache=2048

Platform-Ev=6

PTE-PageSize=8192

CPU_FREQUENCY_HZ=500000000

 

Additional Notes (ASA Reference)

- MemorySizeGB: Alpha OSes (OpenVMS, Tru64, Digital UNIX) expect a minimum   RAM size of 1 GiB or greater for modern platforms. See ASA Vol. I, "Memory  Management" for architectural minimums and page sizing.

- PTE-PageSize: Must be a power of two (4096, 8192, 16384, etc.). 8192 bytes (8 KiB) is canonical for 21164/21264 systems (ASA, Vol. I §5.2.3).

- Platform-Ev: Used by PALcode, SRM/ARC console, and OS for platform-dependent code paths. Ensure this matches the emulated CPU type for correct behavior.

- CPU_FREQUENCY_HZ: Should reflect the closest available clock rate for the target Alpha system and must be consistent with system timing and scheduling code. For ES40/21164, 500 MHz is canonical.
