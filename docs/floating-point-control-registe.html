Floating-Point Control Register (FPCR)

The Floating Point Control Register (FPCR)  is a central architectural register on Alpha AXP systems, controlling all floating-point exception handling, trap enable logic, status ("sticky") bits, rounding modes, and floating-point condition codes.

It is critical for correct delivery of floating-point exceptions, context switching, and OS/PALcode compatibility.

 

Project Implementation:

 

In the ASA-EMulatR, FPCR is modeled by the FpcrRegister class (FpcrRegister.h), which provides:

•Bitfield access for all exception, trap enable, rounding, and status bits

•Canonical set/get methods for each architectural feature

•Strict adherence to Alpha AXP Table 9-1, Section 9.2 (Floating-Point Control Register)

 

Responsibilities:

 

Arithmetic instruction grains:

•Read only: Query trap enable bits and rounding mode during execution.

•Update OptimizedExceptionStatusRegister (not FPCR) for status on each FP operation.

 

PALcode/OS/MT_FPCR:

•Update FPCR: Set/clear trap enables, status bits, and rounding mode.

 

Trap handler:

•May update FPCR status/sticky bits (rare).

•May read/clear sticky bits on exception handling.

 

Bitfields and Controls (Summary):

 

•Exception/Status Flags (bits 0–4): Invalid, Divide-by-Zero, Overflow, Underflow, Inexact

•Trap Enable Flags (bits 5–9): Enable trapping for each corresponding exception

•Rounding Modes (bits 10–13): Dynamic, Minus Infinity, Chopped (toward zero), Plus Infinity

•Sticky Status Bits (bits 15–19): Record if exception occurred (not cleared except by PALcode/OS)

•Reserved and architecture-specific bits (20–23): Reserved operand, invalid PAL function, unaligned access, exception summary

•Floating-Point Condition Codes (FPCC) (bits 24–26): Equal, Less Than, Greater Than

 

Usage in Emulator:

 

Arithmetic Grains:

•Always check FPCR trap enables before raising an FP exception

•Update sticky bits if trap is not delivered (deferred/"sticky" exception)

 

Context Switch:

 

•FPCR is always saved and restored along with processor state

 

Emulator API:

 

FpcrRegister API: Enumerated Cases and Canonical Methods

 

Exception Trap Enable Bits (per exception):

osetTrapInvalidEnable(bool), isTrapInvalidEnabled()

osetTrapDivideByZeroEnable(bool), isTrapDivideByZeroEnabled()

osetTrapOverflowEnable(bool), isTrapOverflowEnabled()

osetTrapUnderflowEnable(bool), isTrapUnderflowEnabled()

osetTrapInexactEnable(bool), isTrapInexactEnabled()

 

Exception Occurred/Status Bits (sticky):

osetInvalidOccurred(bool), hasInvalidOccurred()

osetDivideByZeroOccurred(bool), hasDivideByZeroOccurred()

osetOverflowOccurred(bool), hasOverflowOccurred()

osetUnderflowOccurred(bool), hasUnderflowOccurred()

osetInexactOccurred(bool), hasInexactOccurred()

 

Rounding Modes:

osetRoundingMode(RoundingMode mode)

ogetRoundingMode() (Where RoundingMode is typically an enum: Nearest, PlusInfinity, MinusInfinity, Chopped/Zero)

 

Floating-Point Condition Codes:

osetFPConditionEqual(bool), getFPConditionEqual()

osetFPConditionLessThan(bool), getFPConditionLessThan()

osetFPConditionGreaterThan(bool), getFPConditionGreaterThan()

 

Other Canonical Methods (as typically seen in an FPCR class):

osetDynamicMode(bool), isDynamicMode()

osetStatusSummary(bool), hasStatusSummary()

osetReservedOperandOccurred(bool), hasReservedOperandOccurred()

osetUnalignedAccessOccurred(bool), hasUnalignedAccessOccurred()

 

Bit/Mask Access (generic):

osetFlag(FpcrFlagBit, bool), hasFlag(FpcrFlagBit)

ogetRawValue(), setRawValue(quint64)

oFull Context/Copy Operations:

ocopyFrom(const FpcrRegister& other)

otoString() (for debugging/diagnostics)

oclearStatusBits()

 

Explanation:

•For each trap enable (e.g., Overflow, Underflow, etc.) you have both a setter and a getter.

•For each exception occurred status (sticky), you have both a setter and a getter.

•Rounding mode is typically set and retrieved with an enum.

•FP condition codes (Equal/Less/Greater) are supported with set/get.

•Additional status, reserved, and summary bits may have set/get as well.

•Full flag/mask access supports bitfield logic for architectural emulation.

 

Canonical Trap Model Integration:

•For trap model "Trap Only if Both":

•The grain checks both exception occurred and FPCR trap enable before calling RaiseException.

 

For "Trap Always":

•Only status is checked; FPCR is ignored (unmasked model).

 

For "Status Only":

OptimizedExceptionStatusRegister and/or FPCR sticky bit is set; no trap is delivered.
