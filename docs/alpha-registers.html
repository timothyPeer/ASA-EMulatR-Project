Alpha AXP Register Architecture (Modernized for Unified/128-bit Emulator Design))

The objective of this documentation is to provide a "single source of truth" with respect to Alpha, Tru64, Unix, VAX registers and their respective canonical defined values.

 

General Purpose Integer Registers (GPRs) — 32 × 128-bit Unified

•Width: Exposed as 128 bits (low 64 bits for standard Alpha opcodes, high bits reserved for SIMD and 128-bit operations).

•Storage: All 32 GPRs are stored in a unified array within AlphaProcessorContext and managed by IprBank.

•Usage:

oAll 32 registers are visible in all modes.

oPALcode and OS access the same register bank.

oStandard integer operations: Use only Low64 (bits 0–63).

oSIMD/128-bit operations (future): Will use the full 128 bits.

Floating Point Registers (FPRs) — 32 × 128-bit Unified

•Width: 128 bits, supports S, T, G, D, and (future) X float types. Standard Alpha FPU ops use only Low64.

•Access:

oAll floating-point loads and stores (LDS, LDT, LDG, etc.) access only the low 64 bits.

o128-bit value used for future SIMD or extended IEEE types.

oAliased F31 is not FPCR—FPCR is a separate control register.

SIMD/Vector Registers (Planned Enhancement)

•Future: Will expose a dedicated register bank for vector/SIMD instructions (ASA-SIMD-Extensions).

•Planned Width: 128 bits per lane; multi-lane architecture for future Intel® 64/AVX compatibility.

 

Control and Processor Registers

•PC: 64 bits. Program Counter. Increments by 4 per instruction.

•PS: 64 bits. Processor Status. Mode, IPL, FEN, SIR, condition codes.

•FPCR: 64 bits. Floating Point Control Register.

•All floating-point exception, trap enable, and rounding mode bits as per IEEE 754 and Alpha SRM.

•Managed via FpcrRegister in IprBank.

•UNIQUE: 64 bits. Per-process value (thread libraries).

•LOCK_FLAG: 64 bits. PAL scratch for atomic locks.

•CYCLE_COUNTER: 64 bits. Free-running cycle count.

 

Memory Management Registers (in IPRBank, via MTPR/MFPR)

•TB_ISN/ISA: Instruction TLB insert entries

•TB_DSN/DSA: Data TLB insert entries

•TB_TAG: Tag for probe/refill

•MM_STAT: Fault & TLB miss status

•VA: Faulting virtual address

•DTB_PTE / ITB_PTE: TLB-miss refill PTE buffers

•DTB_ASN / ITB_ASN: ASN registers (split on EV6)

•PCBB: Process Control Block Base

•PTBR: Page Table Base

•ASN: Address Space Number (Current)

•SISR: Software Interrupt Summary

•IER: Interrupt Enable Register

•IPIR: Inter-processor Interrupt Register

 

PAL-Shadow Registers (Hardware Banked)

•On PAL entry, R0–R7, PC, and PS are banked in hardware.

•Not visible outside PAL mode.

•Emulated in context switch/save/restore logic.

 

Performance Monitor Registers (EV6+)

•PERFMON_CTL0-1: Select event sources.

•PERFMON_CNT0-1: 48-bit event counters.

•MISC_CTL: Control (enables BTB, I-cache parity, etc).

 

Console / Firmware Pseudo-Registers

•HWRPB (0x2000_0000): Hardware Restart Parameter Block.

•SCB (0x0000_0000): System Control Block (trap vectors).

•SYS_CTL (0x2004_0000): Console/firmware settings (soft-reset, LEDs, bootstrap, etc).

•(Treated as RAM, but firmware maps as registers.)

Register Design Notes

•All registers are modeled as unified types; 64-bit ops use Low64 on a dt_float data type, but hardware/state is 128-bit wide for future proofing (SIMD, AVX, X-float).

•All GPRs and FPRs are accessible via the AlphaProcessorContext, which delegates to IprBank for control and MMU registers.

•Future: SIMD and vector lanes will be exposed via a dedicated register class, with support for AVX-like and Alpha-specific extensions.
