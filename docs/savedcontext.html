SavedContext- Processor State Snapshot

The SavedContext structure provides a canonical, architecture-compliant snapshot of the entire processor context for use during process context switches, PALcode operations, exception handling, and software save/restore routines in the ASA-EMulatR. It captures all general-purpose registers, floating-point registers, core status/control registers, and relevant OS/firmware context fields in a compact, validated object.

 

Key Features:

•Stores all integer (GPR) and floating-point (FPR, as dt_tfloat) registers (32 each)

•Saves core privileged registers: PS, FPCR, PTBR, ASN, PSL, PALMode, AST, ASTEN, ASTSR, MCES, PCC

•Tracks floating-point exception summary/status flags

•Context-saved bitmask records which fields have been populated for precise context validation and partial saves/restores

•Designed for thread-safe, deep-copy, and stack usage (context stacks, trap frames, etc.)

•Provides utility methods for register set validation, field access, and shape checks

 

Fields and Bitmask:

•intRegs [QVector<quint64>]: All 32 general-purpose integer registers

•fpRegs [QVector<dt_tfloat>]: All 32 floating-point registers (T_Float or double/IEEE emulated)

•m_ps: Processor Status register

•m_fpcr: Floating-Point Control Register

•m_ptbr: Page Table Base Register

•m_asn: Address Space Number

•m_psl: Processor Status Long (for OS/PALcode compatibility)

•m_palMode: Current PALcode operating mode

•m_ast, m_asten, m_astsr: Asynchronous System Trap and related fields (VMS/OS context)

•m_mces: Machine Check Error Summary

•m_pcc: Processor Cycle Counter

•m_fpExceptionFlags: Floating-point exception summary/status

•m_contextSaved: Bitmask indicating which context fields have been set/are valid

 

Context Bitmask:

•Each register or field in the context has a unique bit index.

•The m_contextSaved bitmask records which fields are currently populated—useful for partial saves (e.g., lazy FPU restore) or validating context completeness.

 

Methods and Usage:

•hasSavedContext(): Returns true if all required context fields (per CONTEXT_REQUIRED_MASK) are saved and valid.

•Setters and getters for all registers and fields; each setter updates the contextSaved bitmask.

•clearContextSavedMask(): Resets the bitmask and clears the register state.

•clear(): Zeroes all GPRs/FPRs and resets all fields.

•Static utility: validateShape() — Verifies register array sizes for architectural compliance.

 

Construction and Assignment:

•Constructors ensure 32-entry GPR and FPR arrays for all snapshots.

•Copy and move semantics are safe (deep-copy via QVector).

•Designed for use in per-CPU stacks, context manager, or as a part of process/trap frame structures.

 

Typical Use Cases:

•Process context switch: Save and restore complete CPU state (including integer, FP, and control registers).

•PALcode/OS trap entry and exit: Populate from hardware frame and restore on exception/trap return.

•Exception/interrupt recovery: Save context to software or hardware exception frame.

•Emulator state snapshot: Diagnostic capture or checkpoint.

 

Compliance:

•Fully matches the Alpha AXP architectural context model (see ASA Vol I, Ch. 3, 14).

•Suitable for OpenVMS, Tru64 UNIX, and custom PALcode.

 

Example:

 

// Save processor context

SavedContext ctx;

ctx.setIntRegs(cpu->getIntRegs());

ctx.setFpRegs(cpu->getFpRegs());

ctx.setPS(cpu->readPS());

ctx.setFPCR(cpu->readFPCR());

ctx.setPTBR(cpu->readPTBR());

// ...etc.

// Use hasSavedContext() to verify completeness before switching/restoring

if (!ctx.hasSavedContext()) {

// Error: context incomplete

}
